Google Chrome

  • 选择Chrome版本
  • 无头模式
  • 沙箱

Google Chrome插件允许Travis CI版本在运行时安装Google Chrome。要使用插件，您需要在其上运行构建值得信赖的构建环境或者macOS构建环境。

选择Chrome版本 #

你可以安装stable或者betaChrome版本但您无法选择特定的数字版本。

addons:
  chrome: stable

{：data-file =“。travis.yml”}

请注意，当Google更新时，底层Chrome版本可以从一个版本更改为另一个版本stable要么beta版。

无头模式 #

您可以在中使用Google Chrome无头模式。

沙箱 #

出于安全原因，Google Chrome在运行时无法提供沙盒

基于容器的环境。

在这种情况下，您可能会看到如下错误消息：

30 11 2017 13:35:42.245:ERROR [launcher]: Cannot start Chrome
  [4315:4315:1130/133541.781662:FATAL:setuid_sandbox_host.cc(157)] The SUID sandbox helper binary was found, but is not configured correctly. Rather than run without sandboxing I'm aborting now. You need to make sure that /opt/google/chrome/chrome-sandbox is owned by root and has mode 4755.

或者像这样：

Selenium::WebDriver::Error::UnknownError:
  unknown error: Chrome failed to start: crashed
    (Driver info: chromedriver=2.38.552522 (437e6fbedfa8762dec75e2c5b3ddb86763dc9dcb),platform=Linux 4.14.12-041412-generic x86_64)

要在基于容器的环境中使用Chrome，请传递--no-sandbox国旗chrome可执行文件。

实现此目标的方法因测试框架而异。下面给出一些例子。有关如何添加的详细信息，请参阅工具的文档--no-sandbox旗。

Karma Chrome Launcher #

同Karma的customlauncher插件，你会把它添加到flags数组：

module.exports = function(config) {
  config.set({
    browsers: ['Chrome', 'ChromeHeadless', 'ChromeHeadlessNoSandbox'],

    // you can define custom flags
    customLaunchers: {
      ChromeHeadlessNoSandbox: {
        base: 'ChromeHeadless',
        flags: ['--no-sandbox']
      }
    }
  })
}

水豚 #

使用时水豚使用Ruby，您可以使用适当的标志设置自定义驱动程序：

require 'capybara'
Capybara.register_driver :chrome do |app|
	options = Selenium::WebDriver::Chrome::Options.new(args: %w[no-sandbox headless disable-gpu])

	Capybara::Selenium::Driver.new(app, browser: :chrome, options: options)
end

Capybara.javascript_driver = :chrome

贝哈特 #

使用时贝哈特你应该将选项传递给selenium2 chrome配置：

default:
  extensions:
    Behat\MinkExtension:
      selenium2:
        # This will probably be the same always, if you follow the guide for browsers below.
        wd_host: http://localhost:8643/wd/hub
        capabilities:
          chrome:
            switches:
              - "--headless"
              - "--disable-gpu"
              - "--no-sandbox"
      javascript_session: selenium2
      browser_name: chrome
