Using SourceClear with Travis CI

  • 创建身份验证令牌
  • 设置环境变量

SourceClear

当您将SourceClear添加到TravisCI项目时，您将在每个构建中获得自动安全性分析。您将获得对开源依赖项的完整分析，包括安全漏洞，过时库和许可证报告。

创建身份验证令牌 #

要为Travis-CI设置SourceClear代理，您必须登录SourceClear，然后执行以下步骤：

1。从左侧边栏，选择代理， 然后新代理。

2。在代理设置页面中，选择特拉维斯-CI

3。选择创建身份验证令牌，并将其复制到剪贴板。您将使用它在扫描期间使用SourceClear进行身份验证。

设置环境变量 #

在Travis-CI中设置环境变量基于每个存储库：

1。从Travis-CI环境中选择要扫描的存储库>

更多的选择>设置

2。在Environment Variables页面上，添加SRCCLR_API_TOKEN并将您的身份验证令牌分配给它。确保切换标记的按钮显示值在构建日志*中关闭确保您的令牌保密的位置。

## 配置Travis-CI存储库

要使用SourceClear进行扫描，请将以下内容添加到您的.travis.yml文件：

addons:
    srcclr: true

{：data-file =“。travis.yml”}

如果在扫描期间需要详细输出，则可以添加debug键：

addons:
    srcclr:
        debug: true

{：data-file =“。travis.yml”}

提交这些更改以触发存储库的构建，SourceClear将执行扫描，将结果显示到SourceClear环境。

如果您希望将SourceClear扫描添加到其他存储库，只需将上面的安装和扫描代码添加到其他任何存储库中.travis.yml你想要的文件，以及SRCCLR_API_TOKEN环境变量，您将能够在每个新构建上执行扫描。
