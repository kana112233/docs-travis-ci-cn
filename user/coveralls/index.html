Coveralls

  • 使用Ruby工作服
  • 工作服和私人存储库
  • 使用其他语言的工作服
  • 在Docker构建中使用Coveralls

工作服是一个托管分析工具，提供有关代码覆盖率的统计信息。

配置Travis CI构建以将结果发送到Coverall始终遵循相同的模式：

  1. 将您的存储库添加到Coveralls。
  2. 配置您的构建以安装您正在使用的编程语言的Coveralls库。
  3. 将工作服添加到您的测试套件中。
  4. 如果您使用Travis CI进行私人回购，请添加service_name: travis-pro到你的.coveralls.yml。

我们将在下面的示例中向您展示如何为Ruby执行此操作。

使用Ruby工作服 #

使用Ruby on Travis CI工作服是Coveralls开箱即用的配置之一有文件。

1.将您的存储库添加到Coveralls #

  1. 登录工作服跟你的GitHub上帐户。
  2. 点击添加REPOS在菜单中。
  3. 点击

###

Gemfile：

# ./Gemfile

gem 'coveralls', require: false

Gemfile.lock

3.将工作服添加到您的测试套件中 #

在您之前，将工作服添加到测试套件的顶部require任何应用代码：

# ./spec/spec_helper.rb
# ./test/test_helper.rb
# ..etc..

require 'coveralls'
Coveralls.wear!

完成这三个步骤之后，下次推送提交时，您将能够查找您的提交！

工作服和私人存储库 #

.coveralls.yml：

service_name: travis-pro

{：data-file =“。coveralls.yml”}
