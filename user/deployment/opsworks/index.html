AWS OpsWorks Deployment

Travis CI可以自动部署您的AWS OpsWorks成功构建后的应用程序。

对于最小配置，您需要做的就是将以下内容添加到您的.travis.yml：

deploy:
  provider: opsworks
  access-key-id: ACCESS-KEY-ID
  secret-access-key: SECRET-ACCESS-KEY
  app-id: APP-ID
  region: REGION # (default: us-east-1)

{：data-file =“。travis.yml”}

region不是必需的，但默认为us-east-1。如果您的应用程序位于不同的区域，您将收到类似“OpsWorks服务错误：无法找到ID为xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx的应用程序”的错误。

您可以从中获取AWS Access Key ID和AWS Secret Access Key这里。建议加密您的AWS Secret Access Key。假设你有travis安装客户端，你可以这样做：

travis encrypt SECRET-ACCESS-KEY --add deploy.secret-access-key

你也可以travis工具为您设置一切：

travis setup opsworks

请记住，上面的命令必须在项目目录中运行，因此它可以修改.travis.yml为了你。请注意region不是通过运行生成的travis setup opsworks。

迁移数据库 #

如果要将travis上的rails数据库迁移到AWS OpsWorks，请添加migrate你的选择.travis.yml。

deploy:
  provider: opsworks
  access-key-id: ACCESS-KEY-ID
  secret-access-key: SECRET-ACCESS-KEY
  app-id: APP-ID
  migrate: true

{：data-file =“。travis.yml”}

从中部署分支 #

默认情况下，Travis CI将仅从您的部署主科。

您可以使用显式指定要部署的分支上选项：

deploy:
  provider: opsworks
  access-key-id: ACCESS-KEY-ID
  secret-access-key: SECRET-ACCESS-KEY
  app-id: APP-ID
  on: production

{：data-file =“。travis.yml”}

或者，您也可以将其配置为从所有分支部署：

deploy:
  provider: opsworks
  access-key-id: ACCESS-KEY-ID
  secret-access-key: SECRET-ACCESS-KEY
  app-id: APP-ID
  on:
    all_branches: true

{：data-file =“。travis.yml”}

从Pull Requests触发的构建将永远不会触发部署。

部署构建工件 #

在测试运行之后和部署阶段之前，Travis CI将清除您所做的任何其他文件和更改。

也许这不是你想要的，因为你可能会产生一些应该被部署的工件（想想资产编译）。现在有一个选项可以跳过清理：

deploy:
  provider: opsworks
  access-key-id: ACCESS-KEY-ID
  secret-access-key: SECRET-ACCESS-KEY
  app-id: APP-ID
  skip_cleanup: true

{：data-file =“。travis.yml”}

等待部署 #

默认情况下，构建将在触发OpsWorks后立即继续

部署。要等待部署完成，请使用等待，直到部署

选项：

deploy:
  provider: opsworks
  access-key-id: ACCESS-KEY-ID
  secret-access-key: SECRET-ACCESS-KEY
  app-id: APP-ID
  wait-until-deployed: true

{：data-file =“。travis.yml”}

Travis CI将等待最多10分钟以完成部署，并进行登录

是否成功。

成功部署后更新应用程序设置 #

默认情况下，Travis CI的部署会触发OpsWorks上的部署，但不会触及任何其他配置。要在App Settings中更新修订版，请使用更新的应用程序上，成功选项。另外你必须设置等待，直到部署选项：

deploy:
  provider: opsworks
  access-key-id: ACCESS-KEY-ID
  secret-access-key: SECRET-ACCESS-KEY
  app-id: APP-ID
  wait-until-deployed: true
  update-app-on-success: true

{：data-file =“。travis.yml”}

Travis CI将等待部署返回成功，然后才更新App Settings中的修订。

有条件的释放 #

只有在满足某些条件时才能部署。

看到有条件释放on:。

在部署之前和之后运行命令 #

有时您希望在部署之前或之后运行命令。你可以使用before_deploy和after_deploy这个阶段。只有在Travis CI实际部署时才会触发这些。

before_deploy: "echo 'ready?'"
deploy:
  ..
after_deploy:
  - ./after_deploy_1.sh
  - ./after_deploy_2.sh

{：data-file =“。travis.yml”}
