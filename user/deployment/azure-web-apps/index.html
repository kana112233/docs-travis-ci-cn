Azure Web App Deployment

Travis CI可以自动部署您的Azure Web App成功建造后。

对于最小配置，您需要做的就是启用并将以下内容添加到您的.travis.yml：

deploy:
  provider: azure_web_apps
  username: azure_deployment_user       # If AZURE_WA_USERNAME isn't set
  password: azure_deployment_password   # If AZURE_WA_PASSWORD isn't set
  site: azure_deployment_sitename       # If AZURE_WA_SITE isn't set
  slot: azure_deployment_slotname       # (optional) If AZURE_WA_SLOT isn't set

{：data-file =“。travis.yml”}

不建议您将未部署的Azure部署凭据放入您的.travis.yml。而是使用隐藏的环境变量或加密变量。

要在Repository Settings中定义变量，请确保您已登录，导航到相关存储库，从cog菜单中选择“Settings”，然后单击“Environment Variables”部分中的“Add new variable”。作为Web界面的替代方案，您也可以使用CLIenv命令。

获取部署进度和日志 #

Azure Web App提供程序可以使用以下命令将Azure的部署进度打印到Travis日志verbose选项。但是，如果身份验证失败，Git将打印您的密码（如果您提供正确的用户/密码组合，则不会。）

deploy:
  provider: azure_web_apps
  verbose: true

从中部署分支 #

默认情况下，Travis CI将仅从您的部署主科。

您可以使用显式指定要部署的分支上

deploy:
  provider: azure_web_apps
  on: production

或者，您也可以将其配置为从所有分支部署：

deploy:
  provider: azure_web_apps
  on:
    all_branches: true

从Pull Requests触发的构建将永远不会触发部署。

注意.gitignore #

正如此部署策略所依赖的那样git，请注意部署将

荣誉.gitignore。

如果你的.gitignorefile匹配构建创建的东西，使用

before_deploy改变

它的内容。

在部署之前和之后运行命令 #

有时您希望在部署之前或之后运行命令。你可以使用before_deploy和after_deploy这个阶段。只有在Travis CI实际部署时才会触发这些。

before_deploy: "echo 'ready?'"
deploy:
  ..
after_deploy:
  - ./after_deploy_1.sh
  - ./after_deploy_2.sh

{：data-file =“。travis.yml”}

###

您可能需要将多个分支部署到不同的插槽。您可以将多个提供程序设置为部署到特定插槽。以下配置将部署master分支到myapp-staging插槽和develop分支到myapp-develop插槽。为了使用你需要的插槽。

deploy:
- provider: azure_web_apps
  slot: myapp-staging
- provider: azure_web_apps
  slot: myapp-develop
  on: develop

{：data-file =“。travis.yml”}
