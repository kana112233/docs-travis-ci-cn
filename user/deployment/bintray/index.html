Bintray Deployment

Bintray。

这是一个例子.travis.yml：

deploy:
  provider: bintray
  file: "Path to a descriptor file, containing information for the Bintray upload"
  user: "Bintray user"
  key: "Bintray API key"
  passphrase: "Optional. In case a passphrase is configured on Bintray and GPG signing is used"
  skip_cleanup: true # to upload artifacts created during the build

{：data-file =“。travis.yml”}

加密您的API密钥 #

travis

$ travis encrypt <YOUR BINTRAY API KEY> --add deploy.key


$ travis encrypt ab012cd345678901234e456fa7bc89def01a23b4 --add deploy.key

###

默认情况下，Travis CI将仅从您的部署主

您可以使用显式指定要部署的分支上选项：

deploy:
  ..
  on: production

{：data-file =“。travis.yml”}

deploy:
  ..
  on:
    all_branches: true

有条件的部署 #

只有在满足某些条件时才能部署。

看到有条件释放on:。

在部署之前和之后运行命令 #

有时您希望在部署之前或之后运行命令。你可以使用before_deploy和after_deploy这个阶段。只有在Travis CI实际部署时才会触发这些。

before_deploy: "echo 'ready?'"
deploy:
  ..
after_deploy:
  - ./after_deploy_1.sh
  - ./after_deploy_2.sh

dry_run### 选项

要测试部署配置，您可以添加dry_run: true防止连接

到Bintray服务器：

deploy:
  ..
  dry_run: true

{：data-file =“。travis.yml”}

        ],
    "publish": true
}

包装部分 #

Bintray包装信息。以下信息对于开源项目是必需的：

  • name是Bintray包名
  • repo是Bintray存储库名称
  • subject是Bintray主题，是用户或组织
  • vcs_url是Bintray版本控制系统url，例如github存储库url
  • licenses是个Bintray许可证{：data-proofer-ignore =“”}，这是一个包含至少一个项目的列表。

文件部分 #

配置要上传到Bintray的文件及其上传路径。

您可以定义一组或多组模式。每组包含三种模式：

  • includePattern：Ruby正则表达式形式的模式，指示要上传到Bintray的文件的路径。如果文件位于根目录中，则必须指明相对路径：\./
  • excludePattern： 可选的。 Ruby正则表达式形式的模式，指示要从includePattern指定的文件列表中删除的文件的路径。
  • uploadPattern：Bintray上传路径。路径可以包含$ 1，$ 2，…形式的符号，这些符号将替换为包含模式中定义的捕获组。

在上面的示例中，上载了以下文件：

  • 所有宝石文件位于build/bin/（包括子目录），除了a下的文件do-not-deploy目录。这些文件将被上传到Bintraygems
  • build/docs。这些文件将被上传到Bintraydocs夹。

注意：正则表达式定义为includePattern和excludePattern属性必须用括号括起来。

Debian上传 #

当使用自动索引布局将工件上载到Bintray上的Debian存储库时，Debian分发信息是必需的，必须指定。作为文件闭包的一部分，matrixParams在描述符文件中指定信息，如以下示例所示：

uploadPattern应该尊重bintray自动布局方案。

"files":
    [{"includePattern": "build/bin/(.*\.deb)", "uploadPattern": "pool/main/m/mypackage/$1",
    "matrixParams": {
        "deb_distribution": "vivid",
        "deb_component": "main",
        "deb_architecture": "amd64"}
    }
]

覆盖现有文件 #

如果Bintray存储库中已存在给定名称的工件，则默认情况下不会覆盖该工件。如果要替换现有文件，请定义override键入矩阵属性：

"files":
    [{"includePattern": "build/bin/(myfile.bin)", "uploadPattern": "$1",
    "matrixParams": {
        "override": 1 }
    }
]
