Script deployment

  • 将参数传递给脚本
  • Ruby版本

如果您的部署需要更多自定义而不是after_success方法允许，

使用自定义脚本。

以下示例运行scripts/deploy.sh在…上develop如果构建成功，则存储库的分支。

deploy:
  provider: script
  script: bash scripts/deploy.sh
  on:
    branch: develop

{：data-file =“。travis.yml”}

如果需要运行多个命令，请编写一个可执行的包装脚本来运行它们。争论的script:在脚本中，部署提供程序需要是单个命令。

如果脚本返回非零状态，则考虑部署

失败，构建将被标记为“错误”。

将参数传递给脚本 #

可以将参数传递给脚本部署。

deploy:
  # deploy develop to the staging environment
  - provider: script
    script: bash scripts/deploy.sh staging
    on:
      branch: develop
  # deploy master to production
  - provider: script
    script: bash scripts/deploy.sh production
    on:
      branch: master

{：data-file =“。travis.yml”}

该脚本可以访问所有常见的内容环境变量。

deploy:
  provider: script
  script: bash scripts/deploy.sh production $TRAVIS_TAG
  on:
    tags: true
    all_branches: true

{：data-file =“。travis.yml”}

Ruby版本 #

为了确保部署一致地运行，我们使用的是Ruby版本

预安装在我们所有的构建映像上，这些映像在更新时可能会发生变化。

  • 该travis_internal_ruby函数打印确切的预安装Ruby版本

如果你需要运行一个需要不同的Ruby版本的命令

预先安装的默认值，您需要明确设置它：

deploy:
  provider: script
  script: rvm use $TRAVIS_RUBY_VERSION do script.rb

{：data-file =“。travis.yml”}
