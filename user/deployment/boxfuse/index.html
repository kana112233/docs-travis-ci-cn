Boxfuse Deployment

Travis CI可以自动部署您的成功构建后的应用程序。

对于最小配置，您需要做的就是将以下内容添加到您的.travis.yml：

deploy:
  provider: boxfuse
  user: "YOUR BOXFUSE CLIENT USER"
  secret: "YOUR BOXFUSE CLIENT SECRET"
  payload: "YOUR APPLICATION ARTIFACT (typically a jar, war, tar.gz or zip file)"

{：data-file =“。travis.yml”}

travis encrypt --add deploy.user
travis encrypt --add deploy.secret

或者，您可以使用Travis CI传递您的凭据叫BOXFUSE_USERBOXFUSE_SECRET。您可以使用Travis CI命令行客户端或直接在Travis CI存储库设置UI中定义这些变量。

最后，您还可以通过放置一个完全配置Boxfuseboxfuse.conf文件存储库的根目录。有关配置的更多信息Boxfuse命令行客户端文档。

指定Boxfuse应用程序和图像版本 #

默认情况下，Boxfuse将根据您的有效负载文件的名称自动检测应用程序和版本。您可以像这样覆盖：

deploy:
  provider: boxfuse
  user: "YOUR BOXFUSE CLIENT USER"
  secret: "YOUR BOXFUSE CLIENT SECRET"
  payload: "YOUR APPLICATION ARTIFACT (typically a jar, war, tar.gz or zip file)"
  image: "YOUR BOXFUSE APP AND VERSION (ex.: myapp:1.23)"

{：data-file =“。travis.yml”}

您也可以使用Travis CI环境变量喜欢TRAVIS_BUILD_NUMBER为图像指定版本。例：image: "myapp:$TRAVIS_BUILD_NUMBER"

指定Boxfuse环境 #

默认情况下，Boxfuse将部署到您的test环境。您可以像这样覆盖：

deploy:
  provider: boxfuse
  user: "YOUR BOXFUSE CLIENT USER"
  secret: "YOUR BOXFUSE CLIENT SECRET"
  payload: "YOUR APPLICATION ARTIFACT (typically a jar, war, tar.gz or zip file)"
  env: "YOUR BOXFUSE ENVIRONMENT (default: test)"

{：data-file =“。travis.yml”}

使用备用配置文件 #

你也可以通过放置一个完全配置Boxfuseboxfuse.conf文件存储库的根目录。您可以指定另一个配置文件，如下所示：

deploy:
  provider: boxfuse
  configfile: "YOUR BOXFUSE CONFIGURATION FILE"

{：data-file =“。travis.yml”}

指定自定义参数 #

如果Boxfuse客户此处不包含您需要的功能，您可以使用以下方法将其他参数传递给Boxfuse可执行文件extra_args参数：

deploy:
  provider: boxfuse
  extra_args: "YOUR EXTRA ARGS (ex.: -X)"

{：data-file =“。travis.yml”}

更多的信息 #

去吧Boxfuse网站了解有关Boxfuse的更多信息以及如何配置它。
