Heroku Deployment

  • 部署自定义应用程序名称
  • 部署特定分支
  • 运行命令
  • 部署构建工件

Travis CI可以自动部署您的Heroku的成功构建后的应用程序。

要使用默认配置，请将加密的Heroku api密钥添加到您的.travis.yml：

deploy:
  provider: heroku
  api_key:
    secure: "YOUR ENCRYPTED API KEY"

{：data-file =“。travis.yml”}

如果你有两个Heroku的和特拉维斯CI安装命令行客户端，您可以获取密钥，加密密钥并将其添加到您的密钥.travis.yml通过从项目目录运行以下命令：

travis encrypt $(heroku auth:token) --add deploy.api_key

travis命令默认使用travis-ci.org作为API端点。如果您的构建继续运行travis-ci.com（即使您的存储库是公共的），添加--pro要覆盖此标志的标志：

travis encrypt $(heroku auth:token) --add deploy.api_key --pro

您还可以使用Travis CI命令行设置工具travis setup heroku。

部署自定义应用程序名称 #

默认情况下，我们将尝试使用与存储库相同的名称部署到应用程序。例如，如果从GitHub存储库部署应用程序特拉维斯-CI /特拉维斯聊天如果没有明确指定应用程序的名称，Travis CI将尝试部署到名为的Heroku应用程序特拉维斯聊天。

您可以通过显式设置名称应用选项：

deploy:
  provider: heroku
  api_key: ...
  app: my-app-123

{：data-file =“。travis.yml”}

也可以将不同的分支部署到不同的应用程序：

deploy:
  provider: heroku
  api_key: ...
  app:
    master: my-app-staging
    production: my-app-production

{：data-file =“。travis.yml”}

如果这些应用程序属于不同的Heroku帐户，则必须对API密钥执行相同操作：

deploy:
  provider: heroku
  api_key:
    master: ...
    production: ...
  app:
    master: my-app-staging
    production: my-app-production

{：data-file =“。travis.yml”}

部署特定分支 #

如果您有特定于分支的选项，则为如上所示，Travis CI将自动确定要部署的分支。否则，它只会从你的部署主科。

您还可以使用显式指定要部署的分支上选项：

deploy:
  provider: heroku
  api_key: ...
  on: production

{：data-file =“。travis.yml”}

或者，您也可以将其配置为从所有分支部署：

deploy:
  provider: heroku
  api_key: ...
  on:
    all_branches: true

{：data-file =“。travis.yml”}

从Pull Requests触发的构建将永远不会触发部署。

运行命令 #

在某些设置中，您可能希望在成功部署后在Heroku上运行命令。你可以这样做跑选项：

deploy:
  provider: heroku
  api_key: ...
  run: "rake db:migrate"

{：data-file =“。travis.yml”}

deploy:
  provider: heroku
  api_key: ...
  run:
    - "rake db:migrate"
    - "rake cleanup"

sleep声明在命令之前添加延迟。

###

使用插件，如{：data-proofer-ignore =“”}或Logentries{：data-proofer-ignore =“”}获取通知rake db:migrate或其他命令。

这些加载项具有电子邮件通知系统，可以在Heroku日志中发生某些字符串匹配时触发。例如，如果日志包含“此以及所有后续迁移已取消”，则可以触发电子邮件通知。

###

有时您想在命令之间或之后重新启动Heroku应用程序。您可以通过添加“restart”命令轻松完成此操作：

deploy:
  provider: heroku
  api_key: ...
  run:
    - "rake db:migrate"
    - restart
    - "rake cleanup"

{：data-file =“。travis.yml”}

部署构建工件 #

在测试运行之后和部署之前，Travis CI将清除您所做的任何其他文件和更改。

deploy:
  provider: heroku
  api_key: ...
  skip_cleanup: true

在部署之前和之后运行命令 #

有时您希望在部署之前或之后运行命令。你可以使用before_deploy和after_deploy这个阶段。只有在Travis CI实际部署时才会触发这些。

before_deploy: "echo 'ready?'"
deploy:
  ..
after_deploy:
  - ./after_deploy_1.sh
  - ./after_deploy_2.sh
