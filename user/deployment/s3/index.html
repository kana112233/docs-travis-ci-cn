成功构建后，Travis CI可以自动将您的构建上传到Amazon S3。

对于最小配置，请将以下内容添加到您的.travis.yml：

deploy:
  provider: s3
  access_key_id: "YOUR AWS ACCESS KEY"
  secret_access_key: "YOUR AWS SECRET KEY"
  bucket: "S3 Bucket"

{：data-file =“。travis.yml”}

你可以找到你的AWS访问密钥在您的亚马逊控制台中您应该使用Travis CI命令行加密密钥：

travis encrypt --add deploy.secret_access_key

前面的示例几乎肯定不理想，因为您可能希望上传构建的二进制文件和文档。组skip_cleanup至true防止Travis CI删除您的构建工件。

deploy:
  provider: s3
  access_key_id: "YOUR AWS ACCESS KEY"
  secret_access_key: "YOUR AWS SECRET KEY"
  bucket: "S3 Bucket"
  skip_cleanup: true

{：data-file =“。travis.yml”}

请注意，部署到S3只会将文件添加到存储桶，但不会删除它们。如果您需要删除已弃用的文件，可以在Amazon S3控制台中手动执行。

而不是将您的Amazon S3配置添加到您的.travis.yml您可以在项目目录中运行Travis CI命令行进行设置：

$ travis setup s3

请记住，上面的命令必须在项目目录中运行，因此它可以修改.travis.yml为了你。

S3 ACL通过选项 #

您可以通过以下方式设置上传文件的aclacl选项如下：

deploy:
  provider: s3
  access_key_id: "YOUR AWS ACCESS KEY"
  secret_access_key: "YOUR AWS SECRET KEY"
  bucket: "S3 Bucket"
  skip_cleanup: true
  acl: public_read

{：data-file =“。travis.yml”}

private，public_read，public_read_write，authenticated_read，bucket_owner_read，bucket_owner_full_control。 ACL默认为private。

注意，为了设置acl，桶的政策必须允许这样的操作通过s3:PutObjectAcl行动。

示例策略可能如下所示：

{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Sid": "AllowPublicRead",
            "Effect": "Allow",
            "Principal": {
                "AWS": "PRINCIPAL_ID"
            },
            "Action": [
                "s3:AbortMultipartUpload",
                "s3:DeleteObject",
                "s3:GetObject",
                "s3:GetObjectAcl",
                "s3:PutObject",
                "s3:PutObjectAcl"
            ],
            "Resource": "arn:aws:s3:::BUCKET_NAME/*"
        }
    ]
}

务必根据需要设置本金和资源。

S3 ACL与桶策略 #

为工件设置ACL的另一种方法是通过S3存储桶策略。

此存储桶策略授予公共读取权限：

{
  "Version": "2014-09-25",
  "Statement":[{
    "Sid":"AllowPublicRead",
        "Effect":"Allow",
      "Principal": "*",
      "Action":["s3:GetObject"],
      "Resource":["arn:aws:s3:::"YOUR BUCKET NAME"/*"
      ]
    }
  ]
}

S3桶区 #

默认区域us-east-1在部署到S3时使用。如果您的存储桶托管在其他区域，则使用默认区域进行部署会导致以下错误。

The bucket you are attempting to access must be addressed using the specified endpoint.
Please send all future requests to this endpoint. (AWS::S3::Errors::PermanentRedirect)

这可以通过使用指定桶的区域来解决region组态。例如，此示例使用eu-west-1区域。

deploy:
  provider: s3
  access_key_id: "YOUR AWS ACCESS KEY"
  secret_access_key: "YOUR AWS SECRET KEY"
  bucket: "S3 Bucket"
  skip_cleanup: true
  region: eu-west-1

{：data-file =“。travis.yml”}

仅从一个文件夹部署 #

通常，您不希望将整个项目上传到S3。您可以告诉Travis CI仅将单个文件夹上传到S3。你可以使用local_dir选项这样做。这个例子上传了build您的项目目录到S3：

deploy:
  provider: s3
  access_key_id: "YOUR AWS ACCESS KEY"
  secret_access_key: "YOUR AWS SECRET KEY"
  bucket: "S3 Bucket"
  skip_cleanup: true
  local_dir: build

{：data-file =“。travis.yml”}

部署到特定的S3文件夹 #

通常，您只想上传到特定的S3文件夹。你可以使用upload-dir用于设置S3目标文件夹的选项。此示例上传到travis-builds你的s3桶的文件夹。

deploy:
  provider: s3
  access_key_id: "YOUR AWS ACCESS KEY"
  secret_access_key: "YOUR AWS SECRET KEY"
  bucket: "S3 Bucket"
  skip_cleanup: true
  upload-dir: travis-builds

{：data-file =“。travis.yml”}

部署到S3托管网站： #

要上传到S3托管网站，请使用此模板上传到您的网站。

deploy:
  provider: s3
  access_key_id: "YOUR AWS ACCESS KEY"
  secret_access_key: "YOUR AWS SECRET KEY"
  bucket: "S3 Bucket"
  skip_cleanup: true
  region: "Bucket region"

{：data-file =“。travis.yml”}

请记住，您需要将存储桶设置为具有的ACLpublic任何人都可以看到您的网站。

部署到多个存储桶： #

如果要上传到多个存储桶，可以执行以下操作：

deploy:
  - provider: s3
    access_key_id: "YOUR AWS ACCESS KEY"
    secret_access_key: "YOUR AWS SECRET KEY"
    bucket: "S3 Bucket"
    skip_cleanup: true
  - provider: s3
    access_key_id: "YOUR AWS ACCESS KEY"
    secret_access_key: "YOUR AWS SECRET KEY"
    bucket: "Second S3 Bucket"
    skip_cleanup: true

{：data-file =“。travis.yml”}

在发布之前和之后运行命令 #

有时您希望在发布gem之前或之后运行命令。你可以使用before_deployafter_deploy

    before_deploy: "echo 'ready?'"
    deploy:
      ..
    after_deploy:
      - ./after_deploy_1.sh
      - ./after_deploy_2.sh

{：data-file =“。travis.yml”}

Content-Encoding## 头 #

Content-Encoding。

相关的MIME类型。

要启用此功能，请添加：

deploy:
  provider: s3
  ..
  detect_encoding: true # <== default is false

{：data-file =“。travis.yml”}

如果文件是压缩的gzip要么compress，它将上传

设置charset## 上Content-Type## 头 #

S3可以采用内容类型标头。通常，这也不包括字符集。如果要添加字符集，请添加default_text_charset

deploy:
  provider: s3
  ..
  default_text_charset: 'utf-8'  # Default is ''

##

S3上传可以选择设置Cache-Control和ExpiresHTTP标头。

设置HTTP标头Cache-Control建议浏览器缓存文件。默认为no-cacheno-cache，no-store，max-age=<seconds>，s-maxage=<seconds> no-transform，public，private。

Expires设置缓存对象不再可缓存的日期和时间。默认为未设置。日期必须采用格式YYYY-MM-DD HH:MM:SS -ZONE。

deploy:
  provider: s3
  ..
  cache_control: "max-age=31536000"
  expires: "2012-12-21 00:00:00 -0000"

{：data-file =“。travis.yml”}

设置dot_match标志以上载以句点开头的文件 #

可以将S3上传设置为上传以a开头的所有文件.

deploy:
  provider: s3
  ..
  dot_match: true

{：data-file =“。travis.yml”}

使用S3兼容的对象存储 #

您可以使用与S3兼容的对象存储，例如Digital Ocean Spaces

通过设置endpoint键。

deploy:
  provider: s3
  ..
  endpoint: https://nyc3.digitaloceanspaces.com

{：data-file =“。travis.yml”}
