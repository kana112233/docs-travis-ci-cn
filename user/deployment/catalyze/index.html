Catalyze Deployment

Travis CI可以自动部署到催化后

成功的建设。

在配置之前.travis.yml你需要：

  1. 找到您的Catalyze git remote：

    1. 确保您的催化环境

    2. 通过运行获取git remotegit remote -v{：#remote}来自关联的存储库。
    3. 编辑你的.travis.yml： ``` deploy: provider: catalyze target: “ssh://git@git.catalyzeapps.com:2222/app1234.git”

{：data-file =“。travis.yml”}

2. 设置Catalyst for Travis CI的部署密钥：
   1. 安装Travis CI[命令行客户端](https://github.com/travis-ci/travis.rb)。
   2. 获取Travis CI项目的公共SSH密钥，并通过运行将其保存到文件中

travis pubkey > travis.pub


   3. 使用catalyst中的catalyst命令行客户端将密钥添加为部署密钥

相关的回购。例如：

catalyze deploy-keys add travisci ./travis.pub code-1


哪里`code-1`是您的服务名称。

3. 将Catalyze设置为Travis CI的已知主机：
   1. 通过运行列出已知主机`cat ~/.ssh/known_hosts`。
   2. 查找并复制来自known_hosts的行，其中包含找到的git remote[步骤1](#remote){：data-proofer-ignore =“”}。它看起来像


   3. 更新你的`before_deploy`介入;涉足`.travis.yml`更新`known_hosts`文件：

before_deploy: echo “git.catalyzeapps.com:2222 ecdsa-sha2-nistp256 BBBB12abZmKlLXNo…” » ~/.ssh/known_hosts


{：data-file =“。travis.yml”}


### 部署文件的子集

只部署`build`文件夹，例如，设置`skip_cleanup: true`和

deploy: provider: catalyze target: “ssh://git@git.catalyzeapps.com:2222/app1234.git” skip_cleanup: true path: “build”


{：data-file =“。travis.yml”}

### 在部署之前和之后运行命令

有时您希望在部署之前或之后运行命令。您可以使用

该`before_deploy`和`after_deploy`

如果Travis CI实际部署，则会触发。

before_deploy: “echo ‘ready?’” deploy: .. after_deploy:

  • ./after_deploy_1.sh
  • ./after_deploy_2.sh

```

{：data-file =“。travis.yml”}
