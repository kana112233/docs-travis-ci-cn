RubyGems Deployment

  • 预释放

Travis CI可以自动发布您的Ruby gemRubyGems的成功建造后。

对于最小配置，您需要做的就是将以下内容添加到您的.travis.yml：

deploy:
  provider: rubygems
  api_key: "YOUR API KEY"

{：data-file =“。travis.yml”}

很可能你只想在新版本的时候部署到RubyGems

你的包裹被削减了。为此，您可以告诉Travis CI仅部署

标记的提交，如下所示：

deploy:
  provider: rubygems
  api_key: "YOUR API KEY"
  on:
    tags: true

{：data-file =“。travis.yml”}

如果您在本地标记提交，请记住运行git push --tags确保您的标签上传到GitHub。

您可以通过以下方式检索您的api密钥这些说明。建议加密该密钥。

假设您安装了Travis CI命令行客户端，您可以这样做：

travis encrypt --add deploy.api_key

系统将提示您在命令行中输入api密钥。

你也可以travis工具为您设置一切：

travis setup rubygems

请记住，上面的命令必须在项目目录中运行，因此它可以修改.travis.yml为了你。

预释放 #

您可以让Travis CI创建一个，而不是为每个新版本的gem发布抢鲜对于每个构建。

这使您的gem用户可以选择下载更新，可能更不稳定的gem版本。

要启用此功能，请将以下行添加到您现有的gemspec下面version线：

s.version = "#{s.version}-alpha-#{ENV['TRAVIS_BUILD_NUMBER']}" if ENV['TRAVIS']

20

###

默认情况下，我们将尝试使用与存储库相同的名称来发布gem。例如，如果您从GitHub存储库中释放gem特拉维斯聊天

您可以通过显式设置名称宝石

deploy:
  provider: rubygems
  api_key: ...
  gem: my-gem-123

{：data-file =“。travis.yml”}

也可以将不同的分支发布到不同的宝石：

deploy:
  provider: rubygems
  api_key: ...
  gem:
    master: my-gem
    old: my-gem-old

{：data-file =“。travis.yml”}

如果这些gem属于不同的RubyGems帐户，则必须对API密钥执行相同操作：

deploy:
  provider: rubygems
  api_key:
    master: ...
    old: ...
  gem:
    master: my-gem
    old: my-gem-old

{：data-file =“。travis.yml”}

Gemspec使用 #

如果您愿意，可以指定可以替换选项gemspec选项：

deploy:
    provider: rubygems
    api_key: ...
    gemspec: my-gemspec.gemspec

{：data-file =“。travis.yml”}

###

如上所示，Travis CI将自动确定要发布的分支。否则，它只会从您的主

deploy:
  provider: rubygems
  api_key: ...
  on:
    branch: production

{：data-file =“。travis.yml”}

或者，您也可以将其配置为从所有分支发布：

deploy:
  provider: rubygems
  api_key: ...
  on:
    all_branches: true

{：data-file =“。travis.yml”}

从Pull Requests触发的构建永远不会触发发布。

释放构建工件 #

在测试运行之后和发布之前，Travis CI将清除您所做的任何其他文件和更改。

也许这不是你想要的，因为你可能会生成一些应该被释放的工件。现在有一个选项可以跳过清理：

deploy:
  provider: rubygems
  api_key: ...
  skip_cleanup: true

{：data-file =“。travis.yml”}

有条件的释放 #

只有在满足某些条件时才能部署。

看到有条件释放on:。

宝石必须事先注册 #

请注意，您上传的宝石必须事先注册。

如果在上载它的主机上不存在gem，则部署将失败。

看到这个GitHub问题详情。

在发布之前和之后运行命令 #

有时您希望在发布gem之前或之后运行命令。你可以使用before_deploy和after_deploy这个阶段。只有在特拉维斯CI实际推动发布时才会触发这些。

before_deploy: "echo 'ready?'"
deploy:
  ..
after_deploy:
  - ./after_deploy_1.sh
  - ./after_deploy_2.sh

{：data-file =“。travis.yml”}
