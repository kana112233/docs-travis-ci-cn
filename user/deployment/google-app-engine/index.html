Google App Engine Deployment

Travis CI可以自动部署您的Google App Engine要么托管虚拟机成功构建后的应用程序。

对于最小配置，请将以下内容添加到您的.travis.yml：

deploy:
  provider: gae
  keyfile: "YOUR SERVICE ACCOUNT JSON FILE"
  project: "YOUR PROJECT ID"

{：data-file =“。travis.yml”}

然后去Google Cloud Console仪表板和：

  1. 启用“Google App Engine Admin API”，
  2. 转到“凭据”，单击“添加凭据”和“服务帐户密钥”，最后单击“JSON”以下载您的服务帐户JSON文件。

它是非常建议您在将密钥提交到仓库之前加密密钥。

首先要确保你有Travis命令行工具安装。

然后从终端执行以下命令：

travis encrypt-file client-secret.json --add

该--addflag会自动将解密步骤添加到.travis文件中。

请记住，上面的命令必须在项目目录中运行，因此它可以修改.travis.yml为了你。

可以找到使用Travis加密密钥的更详细说明这里。

要部署的项目 #

默认情况下，将使用与存储库相同的名称部署项目。通常，你会想要明确配置项目用于匹配Cloud控制台中找到的项目ID的选项（请注意，这有时但不总是与项目名称相同）。

您可以通过显式设置项目ID项目选项：

deploy:
  provider: gae
  keyfile: ...
  project: continuous-deployment-demo

{：data-file =“。travis.yml”}

要部署的版本 #

要么是版国旗或默认必须设置选项。如果default为true，则将部署默认版本，即http://your-project-id.appspot.com。如果版而是设置了标志，它将部署到http://version-dot-your-project-id.appspot.com。

从中部署分支 #

默认情况下，Travis只会从您的部署主科。

您还可以使用显式指定要部署的分支上选项：

deploy:
  provider: gae
  keyfile: ...
  project: ...
  on: production

{：data-file =“。travis.yml”}

或者，您也可以将其配置为从所有分支部署：

deploy:
  provider: gae
  keyfile: ...
  project: ...
  on:
    all_branches: true

{：data-file =“。travis.yml”}

从Pull Requests触发的构建将永远不会触发部署。

部署而不提升 #

默认情况下，部署应用程序时，它将被提升为接收所有流量。你可以使用no_promote选项：

deploy:
  provider: gae
  keyfile: ...
  project: continuous-deployment-demo
  no_promote: true

{：data-file =“。travis.yml”}

除此之外，并根据Google Cloud SDK更改日志，“在未来的Cloud SDK版本中，促使新版本接收所有流量的部署将默认停止以前的版本”。

您可以通过设置选项来防止这种情况发生no_stop_previous_version：

deploy:
  provider: gae
  keyfile: ...
  project: continuous-deployment-demo
  no_stop_previous_version: true

{：data-file =“。travis.yml”}

跳过清理 #

许多App Engine应用程序使用果仁将供应商库需求放入目录中，有时需要构建工件或其他

要部署的凭据。如果是这样，您希望避免在部署之前清除工作目录的Travis清理步骤。

deploy:
    provider: gae
    skip_cleanup: true

{：data-file =“。travis.yml”}
