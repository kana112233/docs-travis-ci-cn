加密文件

存储库.travis.yml环境变量，通知设置和部署API密钥。任何人都可以添加这些加密值，但只能由Travis CI读取。存储库所有者不保留任何密钥材料。

请注意，加密环境变量不可用

加密方案 #

Travis CI使用非对称密码术。对于每个已注册的存储库，Travis CI会生成RSA密钥对。 Travis CI将私钥保密，但使每个人都可以使用存储库的公钥。例如，GitHub存储库foo/barhttps://api.travis-ci.org/repos/foo/bar/key。任何人都可以跑travis encrypt对于任何使用存储库的公钥加密参数的存储库。因此，foo/barfoo/bar的私钥，但价值不能被其他任何人解密（甚至连加密者或者“所有者”都没有foo/bar仓库！）。

关于转义某些符号的注意事项 #

当你使用travis encrypt要加密敏感数据，重要的是要注意它

被处理为bash声明。

这意味着您加密的秘密不应该导致错误bash解析它。

数据不完整会导致bash将错误语句转储到日志中

包含部分敏感数据。

因此，你需要逃避特殊字符

例如大括号，括号，反斜杠和管道符号。

例如，您可以输入ma&w!doc如ma\&w\!doc。

并分配字符串6&a(5!1Ab\至FOO：

travis encrypt "FOO=6\\&a\\(5\\!1Ab\\\\"

travis加密字符串FOO=6\&a\(5\!1Ab\\那么bash用于在构建环境中进行评估。

同样，你可以这样做

travis encrypt 'FOO=6\&a\(5\!1AB\\'

通知示例 #

我们想要将campfire通知添加到我们的.travis.yml文件中，但我们不希望公开公开我们的API令牌。

条目应采用以下格式：

notifications:
  campfire:
    rooms: "[subdomain]:[api token]@[room id]"

{：data-file =“。travis.yml”}

对我们来说，这是一个域名：abcxyz @ 14。

我们加密这个字符串

travis encrypt somedomain:abcxyz@14

产生这样的东西

Please add the following to your .travis.yml file:

  secure: "ABC5OwLpwB7L6Ca...."

我们添加到.travis.yml文件中

notifications:
  campfire:
    rooms:
      secure: "ABC5OwLpwB7L6Ca...."

{：data-file =“。travis.yml”}

我们已经完成了。

详细讨论 #

安全var系统采用表单的值{ 'secure' => 'encrypted string' }在（解析的YAML）配置中，用解密的字符串替换它。

所以

notifications:
  campfire:
    rooms:
      secure: "encrypted string"

{：data-file =“。travis.yml”}

变

notifications:
  campfire:
    rooms: "decrypted string"

{：data-file =“。travis.yml”}

而

notifications:
  campfire:
    rooms:
      - secure: "encrypted string"

{：data-file =“。travis.yml”}

变

notifications:
  campfire:
    rooms:
      - "decrypted string"

{：data-file =“。travis.yml”}

在安全env vars的情况下

env:
  - secure: "encrypted string"

{：data-file =“。travis.yml”}

变

env:
  - "decrypted string"

{：data-file =“。travis.yml”}

获取存储库的公钥 #

您可以使用Travis API获取公钥/repos/:owner/:name/key要么

/repos/:id/key端点，例如：

https://api.travis-ci.org/repos/travis-ci/travis-ci/key

你也可以使用travis用于检索所述密钥的工具：

travis pubkey

或者，如果您不在项目目录中：

travis pubkey -r owner/project

注意，travis使用travis.slug在您的项目中确定端点是否存在（使用检查git config --local travis.slug），如果您重命名您的仓库或将您的仓库移动到另一个用户/组织，您可能需要更改它。
