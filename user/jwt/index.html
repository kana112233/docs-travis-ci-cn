JWT addon

  • 第三方服务集成
  • 与JWT Addon集成的第三方服务列表

{： 。警告}

JWT插件已弃用，并且将被弃用

于2018年4月17日停产。

Travis-CI与Sauce Labs等第三方服务之间的整合依赖于此

上加密变量

适用于受信任的分支机构和提交者。

出于安全原因，加密变量不会暴露给不受信任的拉取请求，

因此拉取请求的构建无法访问第三方集成。

JWT插件使用限时身份验证替换加密变量

令牌，暴露于拉请求而没有安全后果。

为此，JWT插件需要在中启用.travis.yml文件，

并且第三方需要与JWT服务集成并允许

基于令牌的身份验证。

### .travis.yml

将加密密钥添加到jwt部分.travis.yml文件。

这可以手动完成或使用travis encrypt命令

特拉维斯加密：

travis encrypt --add addons.jwt SAUCE_ACCESS_KEY=your-access-key

手动：

addons:
  jwt:
     secure: <SAUCE_ACCESS_KEY ENCRYPTED>

{：data-file =“。travis.yml”}

这也可以支持几种服务：

travis encrypt --add addons.jwt SAUCE_ACCESS_KEY=your-access-key THIRDPARTY_SHARED_SECRET=another-key

手动：

addons:
  jwt:
    - secure: <SAUCE_ACCESS_KEY ENCRYPTED>
    - secure: <THIRDPARTY_SHARED_SECRET ENCRYPTED>

{：data-file =“。travis.yml”}

使用加密密钥 #

原始变量名称在Travis CI构建中可用

包含JWT标记而不是原始值的环境变量。

例如，使用以前的配置SAUCE_ACCESS_KEY和

THIRDPARTY_SHARED_SECRET将作为环境变量提供。

这个插件有多安全？ #

JWT令牌仅在90分钟内有效。它以一种安全的方式签名

传输您的秘密信息，而不必担心它被泄露。

故障排除 #

  1. 检查第三方服务是否正常支持的
  2. 联系第三方支持并向他们提供加密令牌（在测试脚本中回显键），并链接到Travis作业。

第三方服务集成 #

第三方服务需要在服务器端实现新的身份验证方法，以便识别和验证JWT令牌。

JWT图书馆 #

JWT.io有完整的清单支持的语言关于一切如何运作的文档

有效载荷 #

用于生成JWT令牌的示例有效负载：

{
  "iss": "Travis CI, GmbH",
  "slug": "saucelabs-sample-test-frameworks/Java-TestNG-Selenium",
  "pull-request": 15,
  "exp": 1470111801,
  "iat": 1470106401
}

哪里：

  • slug将是travis链接slug
  • pull-request将是空的（""）或拉请求整数
  • exp将在令牌到期时（现在+ 5400秒，所以90分钟）
  • iat是在时间（现在）发布的

冒号分隔的字符串包含冒号和JWT之前的用户名

结肠后的标记。用户名用于从中检索用户对象

用户数据库。以下是针对用户执行的功能

对象和用于验证身份验证的令牌。请注意

代码故意与access_key包含的值无关。它没有

无论是否将JWT令牌或访问密钥传递给该函数。

但是，服务提供商必须将JWT auth尝试添加到已经

现有的认证机制。

import jwt

def authenticate(user, access_key):
    """
    user: db object representing user retrieved based on username from HTTP BASIC AUTH
    access_key: access key or JWT token signed using access key (shared secret)
    returns True when authenication validation passed, otherwise False
    """
    # primary auth method
    if user['access_key'] == access_key:
        return True

    # secondary auth attempt using JWT method
    try:
        return bool(jwt.decode(access_key, user['access_key']))
    except (jwt.DecodeError, jwt.ExpiredSignature):
        return False

与JWT Addon集成的第三方服务列表 #

酱汁实验室 #

添加你的SAUCE_USERNAME作为一个正常的环境变量，和你的SAUCE_ACCESS_KEY作为JWT令牌

（看到酱-CONNECT更多细节）：

env:
  - SAUCE_USERNAME=example_username
addons:
  jwt:
    secure: <SAUCE_ACCESS_KEY ENCRYPTED>
