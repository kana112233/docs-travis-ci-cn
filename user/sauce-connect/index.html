Using Sauce Labs with Travis CI

  • 设置Sauce Connect
  • 其他选项

Travis CI与之融为一体酱汁实验室，一个浏览器和

移动测试平台。例如，它与Selenium很好地集成。

集成会自动设置开始测试所需的隧道

用它。为此，它使用Sauce Connect。

请注意，由于安全限制，Sauce Labs插件无法提供

请求构建，除非您使用JWT Addon。

设置Sauce Connect #

酱汁连接安全代理Sauce之间的浏览器流量

Labs的基于云的VM和您的本地服务器。 Connect使用端口443和80

与Sauce的云沟通。如果您正在使用Sauce Labs作为Selenium

测试，这使得连接到您的网络服务器变得更加容易。

第一，注册与酱汁实验室如果你还没有（它是

自由对于开源项目），并从您的。获取您的访问密钥

帐户页面。完成后，将其添加到.travis.yml

文件：

addons:
  sauce_connect:
    username: "Your Sauce Labs username"
    access_key: "Your Sauce Labs access key"

{：data-file =“。travis.yml”}

如果您不希望在存储库中公开访问您的访问密钥，那么您就是

可以加密它travis encrypt "your-access-key"（看到加密密钥

有关加密的更多信息，并添加拉请求安全安全（请参阅JWT Addon。

字符串如下：

addons:
  sauce_connect:
    username: "Your Sauce Labs username"
  jwt:
    secure: "The secure string output by `travis encrypt SAUCE_ACCESS_KEY=Your Sauce Labs access key`"

{：data-file =“。travis.yml”}

你也可以添加username和access_key作为环境变量，如果你

命名他们SAUCE_USERNAME和SAUCE_ACCESS_KEY， 分别。在这种情况下，

您需要添加到.travis.yml文件中的所有内容是：

addons:
  sauce_connect: true

{：data-file =“。travis.yml”}

为了允许多个隧道同时打开，Travis CI打开了一个

酱汁连接识别隧道。确保你发送

该TRAVIS_JOB_NUMBER打开连接时的环境变量

tunnel-identifier，

这看起来将取决于您正在使用的客户端库

Ruby的硒的webdriver绑定：

caps = Selenium::WebDriver::Remote::Capabilities.firefox({
  'tunnel-identifier' => ENV['TRAVIS_JOB_NUMBER']
})
driver = Selenium::WebDriver.for(:remote, {
  url: 'http://username:access_key@ondemand.saucelabs.com/wd/hub',
  desired_capabilities: caps
})

其他选项 #

有时您可能需要将其他选项传递给Sauce Connect。目前

支持的参数是

  • direct_domains
  • no_ssl_bump_domains
  • tunnel_domains

例如，您可能需要--direct-domains选项以防万一[

无法使用Sauce Connect](https://support.saucelabs.com/hc/en-us/articles/225267468–Bad-Gateway-or-Security-Warnings-When-Using-Sauce-Connect-for-Testing-Web-Applications-over-HTTPS)：

addons:
  sauce_connect:
    username: "Your Sauce Labs username"
    direct_domains: example.org,*.foobar.com
  jwt:
    secure: "The secure string output by `travis encrypt SAUCE_ACCESS_KEY=Your Sauce Labs access key`"

{：data-file =“。travis.yml”}
