Installing Dependencies

  • 在标准基础结构上安装包
  • 在多个操作系统上安装依赖项
  • 从源安装项目

在标准基础结构上安装包 #

安装未包含在标准中的Ubuntu软件包精确，before_install你的一步.travis.yml：

before_install:
  - sudo apt-get install -y libxml2-dev

{：data-file =“。travis.yml”}

默认情况下，apt-get update没有自动运行。如果你想更新apt-get自动在每个构建上，有两种方法可以做到这一点。首先是运行apt-get update显然在before_install步：

before_install:
  - sudo apt-get update
  - sudo apt-get install -y libxml2-dev

{：data-file =“。travis.yml”}

第二种方式是使用APT插件：

before_install:
  - sudo apt-get install -y libxml2-dev
addons:
  apt:
    update: true

{：data-file =“。travis.yml”}

别跑apt-get upgrade在您的构建中，它下载最多500MB的包，并显着延长您的构建时间。 使用-yapt-get的参数假设为每个apt-get提示符的答案。

从自定义APT存储库安装包 #

对于某些软件包，您可能会找到一个现有的存储库，默认情况下尚未在我们的构建环境中设置。您可以轻松添加自定义存储库和Launchpad PPA作为构建的一部分。

例如，从ubuntu-toolchain ppa安装gcc

before_install:
  - sudo add-apt-repository ppa:ubuntu-toolchain-r/test -y
  - sudo apt-get update -q
  - sudo apt-get install gcc-4.8 -y

{：data-file =“。travis.yml”}

对于未在Launchpad上托管的存储库，您还需要添加GnuPG密钥。

如果您以这种方式安装软件包，请确保为您的环境下载正确的版本。

此示例将适用于Ubuntu 12.04的Varnish 3.0的APT存储库添加到本地可用的APT源列表中，然后安装varnish包。

before_script:
  - curl http://repo.varnish-cache.org/debian/GPG-key.txt | sudo apt-key add -
  - echo "deb http://repo.varnish-cache.org/ubuntu/ precise varnish-3.0" | sudo tee -a /etc/apt/sources.list
  - sudo apt-get update -qq
  - sudo apt-get install varnish -y

{：data-file =“。travis.yml”}

在没有APT存储库的情况下安装软件包 #

对于某些项目，可能有Debian / Ubuntu包，但没有相应的APT存储库。这些仍然易于安装，但需要额外的下载步骤。

如果您以这种方式安装软件包，请确保为您的环境下载正确的版本。

假设您的项目需要pngquant工具来压缩PNG文件，以下是如何下载和安装.deb文件：

before_install:
  - wget http://pngquant.org/pngquant_1.7.1-1_i386.deb
  - sudo dpkg -i pngquant_1.7.1-1_i386.deb

{：data-file =“。travis.yml”}

使用APT Addon安装软件包 #

您也可以使用APT插件安装软件包和源代码，而无需运行apt-get你的命令before_install脚本。

如果您的要求超出正常安装范围，请使用上述其他方法。

添加APT源 #

要添加APT源，您可以使用以下三种类型的条目之一：

  1. 别名中定义的别名来源安全列表
  2. sourceline将被添加到的键值对/etc/apt/sources.list
  3. 当APT源需要GPG密钥时，您可以使用key_url除了对sourceline。

以下代码段显示了这三种类型的APT源

addons:
  apt:
    sources:
    - deadsnakes
    - sourceline: 'ppa:ubuntu-toolchain-r/test'
    - sourceline: 'deb https://packagecloud.io/chef/stable/ubuntu/precise main'
      key_url: 'https://packagecloud.io/gpg.key'

{：data-file =“。travis.yml”}

添加APT包 #

列出APT包下的addons.apt.packages键：

addons:
  apt:
    packages:
    - cmake
    - time

{：data-file =“。travis.yml”}

注意：将APT源和包一起使用时，您需要进行操作

确保它们位于YAML文件中的相同密钥空间下。例如

addons:
  apt:
    sources:
    - ubuntu-toolchain-r-test
    packages:
    - gcc-4.8
    - g++-4.8

{：data-file =“。travis.yml”}

注意：如果apt-get install失败，构建标记为错误。

使用Snap Addon安装Snap软件包 #

你可以安装卡使用我们的Xenial图像的包：

dist: xenial

{：data-file =“。travis.yml”}

Ubuntu Snap商店提供许多由上游直接维护的软件包

开发人员，通常使用的版本比Apt存档中的版本更新。

您可以将快照指定为快照规范数组，每个规格都是

两种可能的形式：

  1. 快照的名称，将传递给snap install无

额外的旗帜。例如，

dist: xenial
addons:
  snaps:
    - hugo

{：data-file =“。travis.yml”} 这导致：

$ sudo snaps install hugo
  1. 指定应如何安装快照的地图。可能的关键是：

name，confinement，和channel。

该confinement密钥将用于添加--classic要么--devmode旗，

和channel将被传递给--channel旗。

例如，

dist: xenial
addons:
  snaps:
    - name: aws-cli
      confinement: classic # or devmode
      channel: latest/edge # will be passed to --channel flag

{：data-file：“。travis.yml”} 这导致：

$ sudo snaps install aws-cli --classic --channel=latest/edge

confinementchannel

从水龙头安装 #

Homebrew支持从第三方存储库安装桶和包，你可以使用Homebrew插件。

例如，Homebrew维护了某些旧版本的旧版本homebrew/cask-versions。如果要在安装了Java 10的映像上安装Java 8，可以添加该漏洞，然后安装java8木桶：

osx_image: xcode10
addons:
  homebrew:
    taps: homebrew/cask-versions
    casks: java8

{：data-file =“。travis.yml”}

使用Brewfile #

在引擎盖下，Homebrew插件通过创建一个~/.Brewfilebrew bundle --globalBrewfile签入您的项目。通过传递brewfile: true，插件将寻找一个Brewfile在项目的根目录中：

addons:
  homebrew:
    brewfile: true

{：data-file =“。travis.yml”}

如果Brewfile位于其他位置，您还可以提供路径。

addons:
  homebrew:
    brewfile: Brewfile.travis

如果你正在运行brew

rvm use 2.3 --install --binary

在多个操作系统上安装依赖项 #

如果您在Linux和macOS上进行测试，则可以同时使用APT插件和Homebrew插件。每个插件只能在适当的平台上运行：

addons:
  apt:
    packages: foo
  homebrew:
    packages: bar

{：data-file =“。travis.yml”}

如果您手动安装软件包，请使用$TRAVIS_OS_NAME变量为每个操作系统单独安装依赖项：

install:
  - if [ $TRAVIS_OS_NAME = linux ]; then sudo apt-get install foo; else brew install bar; fi

{：data-file =“。travis.yml”}

从源安装项目 #

某些依赖项只能从源包安装。构建可能需要更新版本或不可用作Ubuntu软件包的工具或库。

您可以通过运行脚本来处理安装过程，轻松地在.travis.yml中包含构建步骤，或者这是推荐的方法。

这是一个从二进制包安装CasperJS的简单示例：

before_script:
  - wget https://github.com/n1k0/casperjs/archive/1.0.2.tar.gz -O /tmp/casper.tar.gz
  - tar -xvf /tmp/casper.tar.gz
  - export PATH=$PATH:$PWD/casperjs-1.0.2/bin/

{：data-file =“。travis.yml”}

请注意，当您更新时$PATH环境变量，该部分无法移动到shell脚本中，因为它只会更新运行脚本的子进程的变量。

要从源安装某些内容，您可以执行类似的步骤。下面是一个下载，编译和安装protobufs库的示例。

install:
  - wget https://protobuf.googlecode.com/files/protobuf-2.4.1.tar.gz
  - tar -xzvf protobuf-2.4.1.tar.gz
  - pushd protobuf-2.4.1 && ./configure --prefix=/usr && make && sudo make install && popd

{：data-file =“。travis.yml”}

可以将这三个命令提取到shell脚本中，让我们为它命名install-protobuf.sh：

#!/bin/sh
set -ex
wget https://protobuf.googlecode.com/files/protobuf-2.4.1.tar.gz
tar -xzvf protobuf-2.4.1.tar.gz
cd protobuf-2.4.1 && ./configure --prefix=/usr && make && sudo make install

一旦将其添加到存储库，您就可以从.travis.yml运行它：

before_install:
  - ./install-protobuf.sh

{：data-file =“。travis.yml”}

请注意，第一个版本使用pushd和popd确保之后install部分完成后，工作目录将返回其原始值。这在shell脚本中不是必需的，因为它在子shell中运行，因此不会改变原始工作目录。
