Uploading Artifacts on Travis CI

Travis CI可以自动将您的构建工件上传到Amazon S3

工作结束后after_script相。

请注意，工件插件不适用于拉取请求构建。

对于最小配置，请将以下内容添加到您的.travis.yml：

addons:
  artifacts: true

{：data-file =“。travis.yml”}

并在存储库设置中添加以下环境变量：

ARTIFACTS_KEY=(AWS access key id)
ARTIFACTS_SECRET=(AWS secret access key)
ARTIFACTS_BUCKET=(S3 bucket name)

该地区默认为us-east-1。对于任何其他地区，要么定义ARTIFACTS_REGION环境变量或将其添加到工件配置键下：

addons:
  artifacts:
    s3_region: "us-west-1" # defaults to "us-east-1"

{：data-file =“。travis.yml”}

您可以找到您的AWS Access密钥这里。

部署特定路径 #

上传到S3的默认路径可通过以下方式找到git ls-files -o在

为了找到未跟踪的git工作副本中的任何文件。

如果需要上传任何其他路径，可以通过指定它们

该addons.artifacts.paths关键是这样的：

addons:
  artifacts:
    # ...
    paths:
    - $(git ls-files -o | tr "\n" ":")
    - $(ls /var/log/*.log | tr "\n" ":")
    - $HOME/some/other/thing.log

{：data-file =“。travis.yml”}

或者作为存储库设置中的环境变量：

# ':'-delimited paths, e.g.
ARTIFACTS_PATHS="./logs:./build:/var/log"

请记住，在上面的例子中，冒号（:）用作a

分隔符，表示文件名不能包含此字符。

工作目录 #

如果您想从特定目录上传文件，可以通过设置更改工作目录addons.artifacts.working_dir。

addons:
  artifacts:
    # ...
    working_dir: out

{：data-file =“。travis.yml”}

目标路径 #

默认情况下，工件将上载到存储桶中的路径

被定义为/${TRAVIS_REPO_SLUG}/${TRAVIS_BUILD_NUMBER}/${TRAVIS_JOB_NUMBER}。

您可以使用以下命令在构建时更改上载路径target_paths

键，例如：

addons:
  artifacts:
    target_paths:
    - /$TRAVIS_OS_NAME/$((lsb_release -rs 2>/dev/null || sw_vers -productVersion) | grep --only -E '^[0-9]+\.[0-9]+')

{：data-file =“。travis.yml”}

调试 #

如果您想了解更多关于工件插件的详细信息

做，设置addons.artifacts.debug任何非空的都会变成

关于调试日志记录

addons:
  artifacts:
    # ...
    debug: true

{：data-file =“。travis.yml”}

或者将其定义为存储库设置环境变量，或者在env.global部分：

ARTIFACTS_DEBUG=1

Travis CI Artifact Uploader #

对于更复杂的工件上传，您可以使用神器上传工具默认情况下，它安装在您的构建VM上。
