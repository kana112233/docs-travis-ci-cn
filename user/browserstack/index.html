Using BrowserStack with Travis CI

  • 设置BrowserStack
  • 其他选项

Travis CI与之融为一体BrowserStack，跨浏览器和真实设备

基于网络的测试平台。 BrowserStack可用于通过框架进行交互式和自动化测试

像Selenium，Karma和其他人。

此加载项自动设置BrowserStack Local这允许您使用BrowserStack云测试您的私人服务器以及公共URL。为此，它使用了BrowserStackLocal二进制文件为您的构建平台。

BrowserStack Local在Travis构建容器/ VM之间建立安全连接

和BrowserStack服务器。本地测试还支持防火墙，代理和Active Directory。

设置安全连接后，所有URL都可以开箱即用，包括您的网络服务器，本地文件夹以及

使用HTTPS的网址。

设置BrowserStack #

如果您尚未注册，请注册BrowserStack帐户;它的

自由用于开源项目。注册后，请从中获取用户名和访问密钥

该帐号设定页。您需要配置用户名和访问密钥才能配置.travis.yml

你的项目的文件。

选择是要将访问密钥存储为纯文本还是以安全/加密形式存储。对于我们推荐的开源项目

以安全的形式存储访问密钥，以便拉取请求不能使用存储在您的密钥中的密钥.travis.yml。

有关更多信息，请参阅拉请求页面。

加密的访问密钥 #

加密您的访问密钥以供使用.travis.yml您可以使用travis encrypt "your BrowserStack access key"。

您需要安装travis cli才能执行此操作（请参阅加密密钥更多细节）。

加密访问密钥后，您可以添加安全字符串：

addons:
  browserstack:
    username: "Your BrowserStack username"
    access_key:
      secure: "The secure string output of `travis encrypt`"

{：data-file =“。travis.yml”}

纯文本访问密钥 #

要以纯文本格式存储访问密钥，请将以下配置添加到您的.travis.yml文件：

addons:
  browserstack:
    username: "Your BrowserStack username"
    access_key: "Your BrowserStack access key"

{：data-file =“。travis.yml”}

###

附加组件会总是为每个创建的本地连接创建本地标识符。如果您正在使用Selenium

在测试框架中，必须将本地标识符添加到Selenium功能中。

本地标识符作为环境变量公开BROWSERSTACK_LOCAL_IDENTIFIER

硒的webdriver：

require 'rubygems'
require 'selenium-webdriver'

# Input capabilities
caps = Selenium::WebDriver::Remote::Capabilities.new
caps['browserstack.local'] = 'true'
caps['browserstack.localIdentifier'] = ENV['BROWSERSTACK_LOCAL_IDENTIFIER']
# Add other capabilities like browser name, version and os name, version
...

driver = Selenium::WebDriver.for(:remote,
  :url => "http://USERNAME:ACCESS_KEY@hub-cloud.browserstack.com/wd/hub",
  :desired_capabilities => caps)

本地标识符是必不可少的矩阵构建。由于travis中的矩阵构建可以运行

在同一个VM中，我们需要在启动连接时添加Local Identifier以确保正确的本地隧道

得到正确的要求。

应用上传 #

构建后，将您的应用程序上传到BrowserStack服务器。应在安装步骤中构建应用程序，并且必须在脚本步骤中运行测试脚本。要上传应用程序，请在.travis.yml文件中配置应用程序的路径：

install:
  - "Build script for the app"
script:
  - "Test script"
addons:
  browserstack:
    username: "Your BrowserStack username"
    access_key: "Your BrowserStack access key"
    app_path: "path to your app file"

{：data-file =“。travis.yml”}

将应用程序上传到BrowserStack服务器后，将在环境变量中设置生成的应用程序IDBROWSERSTACK_APP_ID。您可以使用它在测试中设置Appium功能。

caps['app'] = ENV['BROWSERSTACK_APP_ID']

查看BrowserStack Android示例应用程序.travis.yml文件。

其他选项 #

代理 #

本地测试还允许您设置代理主机，端口，用户名和密码

通过它解决所有网址：

addons:
  browserstack:
    username: "Your BrowserStack username"
    access_key:
      secure: "The secure string output of `travis encrypt`"
    proxyHost: "Proxy server host"
    proxyPort: "Proxy server port"
    proxyUser: "User to use when accessing proxy server"
    proxyPass: "Password to use when accessing proxy server"

{：data-file =“。travis.yml”}

###

  • **：如果设置为true，则将通过Travis CI容器/ VM解析所有网络流量。
  • 只要：限制对指定本地服务器和/或文件夹的本地测试访问。

样品用法，

addons:
  browserstack:
    username: "Your BrowserStack username"
    access_key:
      secure: "The secure string output of `travis encrypt`"
    forcelocal: true
    only: dev.example.com,80,0,*.example.org,80,0

{：data-file =“。travis.yml”}

的格式**flag是，“主机模式，主机端口，SSL标志为True（1）/ False（0）”并重复。
