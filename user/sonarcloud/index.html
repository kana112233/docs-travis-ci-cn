Using SonarCloud with Travis CI

  • 要求
  • 使用SonarQube扫描仪检查代码
  • 分析内部拉动请求
  • 即将到来的改进
  • 不推荐的功能
  • SonarQube用户请注意

SonarCloud是一种由云提供的服务SonarSource并基于。 SonarQube是一个广泛采用的开源平台，可以持续检查源代码的质量，并检测20多种不同语言的错误，漏洞和代码异味。

请参考SonarQube文档有关如何配置不同扫描仪的更多详细信息。

要求 #

您正在使用以下两种环境之一：

  • 带有JVM VM映像的CI环境- 例如：
language: java

{：data-file =“。travis.yml”}

  • 诚信的CI环境：
dist: trusty

{：data-file =“。travis.yml”}

使用SonarQube扫描仪检查代码 #

在检查代码之前，您需要：

  1. 创建用户身份验证令牌为您在SonarCloud上的帐户。
  2. 加密此令牌travis encrypt abcdef0123456789或定义SONAR_TOKEN在你的存储库设置
  3. 找出哪个SonarCloud.io组织你想推动你的项目并获得它的关键
  4. 创建一个sonar-project.properties您的项目的文件（请参阅文件）。

然后将以下行添加到您的.travis.yml文件触发分析：

addons:
  sonarcloud:
    organization: "sonarcloud_organization_key" # the key of the org you chose at step #3
    token:
      secure: "*********" # encrypted value of your token
script:
  # other script steps might be done before running the actual analysis
  - sonar-scanner

{：data-file =“。travis.yml”}

请看看了解有关此标准用例的更多信息。

适用于Maven的SonarQube扫描仪 #

很多Java项目都是用Maven构建的。要在Maven构建中添加SonarCloud检查，请将以下内容添加到您的.travis.yml文件：

addons:
  sonarcloud:
    organization: "sonarcloud_organization_key" # the key of the org you chose at step #3
    token:
      secure: "*********" # encrypted value of your token
script:
  # the following command line builds the project, runs the tests with coverage and then execute the SonarCloud analysis
  - mvn clean org.jacoco:jacoco-maven-plugin:prepare-agent install sonar:sonar

{：data-file =“。travis.yml”}

请看看基于Maven的实例示例项目了解这个用例的更多信息。

分析内部拉动请求 #

SonarCloud可以检查存储库的内部拉取请求，并在发现问题的每一行上写评论。

内部拉请求。换句话说，不会检查从fork构建的拉取请求。

要激活对拉取请求的分析，您需要安装SonarCloud应用程序在您的GitHub组织上。

请注意，如果您在GitHub应用程序之前使用SonarCloud并因此在项目上配置了GitHub令牌，那么您现在应该从项目的“管理>常规设置>请求”页面中删除这些令牌。

即将到来的改进 #

此附加组件的未来版本将提供以下功能：

  • 支持外部拉取请求。

不推荐的功能 #

如果您是SonarCloud的长期用户，您可能会在其中输入以下条目.travis.yml文件：

  • “github_token”

如果是这种情况，您将在日志中收到警告，告诉您将很快删除此行为。你应该摆脱你的那些条目.travis.yml文件以受益于SonarCloud的最新功能。

SonarQube用户请注意 #

如果您熟悉SonarQube，您可能会想要处理一些属性亲属分支分析（例如：sonar.branch.name）和/或拉动请求分析（例如：sonar.pullrequest.key）。

这些属性完全没用，SonarCloud附加组件根据分析类型为您管理它们。
