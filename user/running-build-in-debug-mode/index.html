Running Build in Debug Mode

  • 启用调试模式
  • 在调试模式下重新启动作业
  • 连接到在调试模式下运行的作业
  • 在调试VM内部时要做的事情
  • 已知的问题

如果您在解决复杂的构建错误时遇到问题，或者您怀疑存在错误

您当地的发展环境和地区之间的显着差异

在Travis CI构建环境中，您可以在调试模式下重新启动构建

获取对虚拟机或容器的shell访问权限。

启用调试模式 #

私有存储库默认启用调试模式，不需要进行任何更改。

要限制对调试的访问，请仅授予用户权限读取权限到repo，并使用fork + PR工作流程。

对于公共存储库，我们必须在存储库的基础上启用它。 要为您的公共存储库启用调试，请发送电子邮件至

support@travis-ci.com并告诉我们您要激活哪些存储库。

在调试模式下重新启动作业 #

“调试版本”或“调试作业”按钮位于右上角

私有存储库的构建和作业页面。对于开源存储库，

此按钮不可用，您需要使用API​​调用。

通过API在调试模式下重新启动作业 #

要通过API在调试模式下重新启动作业，请发送一个POST要求工作debug端点。

需要通过添加您的请求来验证此请求Travis CI API令牌

到了Authorization头。您可以在Travis CI配置文件页面中找到您的API令牌

对于公共项目。

（注意文字token必须在实际授权令牌之前出现。）

$ curl -s -X POST \
  -H "Content-Type: application/json" \
  -H "Accept: application/json" \
  -H "Travis-API-Version: 3" \
  -H "Authorization: token ********************" \
  -d "{\"quiet\": true}" \
  https://api.travis-ci.com/job/${id}/debug

由于公共存储库不显示“调试”按钮，因此这是重新启动构建的唯一方法

在公共存储库的调试模式下。

请注意，如果您仍在使用travis-ci.org你需要使用https://api.travis-ci.org/job/${id}/debug在上一个命令中。

遗留存储库 #

尚未迁移到的公共存储库

travis-ci.com要求您按如下方式提出API请求（星号应该是

替换为来自travis-ci.org的令牌）：

$ curl -s -X POST \
  -H "Content-Type: application/json" \
  -H "Accept: application/json" \
  -H "Travis-API-Version: 3" \
  -H "Authorization: token ********************" \
  -d "{\"quiet\": true}" \
  https://api.travis-ci.org/job/${id}/debug

找到工作ID #

该${id}是作业ID，而不是构建ID。例如，ID248927956在URL中https://travis-ci.org/travis-ci/docs-travis-ci-com/builds/248927956是构建ID。要获取相应的作业ID，请单击查看配置该页面上的按钮，URL将变为https://travis-ci.org/travis-ci/docs-travis-ci-com/jobs/248927957/config248927957。或者，您可以获取与构建ID对应的作业ID。

将调试会话的输出回应到日志 #

要显示连接到调试中运行的VM的任何命令输入和输出

模式，传递一个空的JSON对象（{}）而在有效载荷中。

连接到在调试模式下运行的作业 #

当作业在调试模式下启动时，它将完成初始步骤，包括设置语言

运行时和缓存（如果适用）并安装调试实用程序。

当它准备好连接时，您将看到：

Debug build initiated by BanzaiMan
Setting up debug tools.
Preparing debug sessions.
Use the following SSH command to access the interactive debugging environment:
ssh DwBhYvwgoBQ2dr7iQ5ZH34wGt@ny2.tmate.io
This build is running in quiet mode. No session output will be displayed.
This debug build will stay alive for 30 minutes.

运行了ssh上面的命令会让你进入实时虚拟机。

在调试模式下运行的作业将具有TRAVIS_DEBUG_MODE环境变量调成true。

安全考虑 #

这时，除了之外没有任何身份验证ssh命令显示在

日志。

你可以选择分享ssh与另一个可能的人的命令

能够帮助您调试构建。

但是，这也意味着任何可以在调试时看到命令的人

VM现场可以连接到调试VM并可能查看您的秘密。

这就是默认情况下公共功能不可用的原因

库。

请注意另一个ssh连接开始，即新连接

将附加到您已经在的会话中。

当您已连接到实时调试VM并且新用户连接时

使用相同的VMssh命令，您可以看到新用户正在做什么。

例如，如果新用户运行env列出所有环境变量，

您也可以在终端上看到输出。

这清楚地表明您的秘密存在风险，而且您

应采取措施减轻任何可能的损害。

如果您怀疑是恶意活动，可以关闭所有窗口或取消

Web UI上的调试作业断开攻击者的连接。

“权限被拒绝”错误消息 #

进入后ssh命令，您可能会看到如下错误消息：

$ ssh DwBhYvwgoBQ2dr7iQ5ZH34wGt@ny2.tmate.io
The authenticity of host 'ny2.tmate.io (104.236.9.236)' can't be established.
ECDSA key fingerprint is c7:a1:51:36:d2:bb:35:4b:0a:1a:c0:43:97:74:ea:42.
Are you sure you want to continue connecting (yes/no)? yes
Warning: Permanently added 'ny2.tmate.io,104.236.9.236' (ECDSA) to the list of known hosts.
Permission denied (publickey).

如果看到这个，请运行该命令ssh-keygen（完成提示），然后再试一次。

在调试VM内部时要做的事情 #

进入调试VM后，您就可以进行探索了。

你在tmate会议，在这一点上

在哪里你的before_install（即使没有定义）阶段即将到来

开始。

方便bash### 功能 #

为方便起见，定义了各种构建阶段bash函数命名

travis_run_*（例如。，travis_run_before_install，travis_run_install）。

运行这些构建阶段以观察您的构建：

travis_run_before_install
travis_run_install
travis_run_before_script
travis_run_script
travis_run_after_success
travis_run_after_failure
travis_run_after_script

基本tmate### 特征 #

tmate是一个分叉tmux。

您的调试VM会话使用默认配置;因此，命令前缀是

ctrl-b。

在此调试会话中，您可以使用功能的子集。

创建一个新窗口 #

ctrl-b c

第一个窗口的索引为0.因此，重复命令将为您提供窗口

1,2等等。

切换到不同的窗口 #

ctrl-b 0

这会将会话的焦点切换到索引为0的窗口。

你可以替代0使用任何有效索引切换到该窗口。

如果要在其中运行长时间运行的进程，则在窗口之间切换会很有帮助

一个窗口，同时在另一个窗口中查看调试VM。

向上和向下滚动终端历史记录 #

ctrl-b [

进入日志滚动模式。在这里，您可以使用箭头键移动光标以完成

记录历史。

按q退出日志滚动模式。

退出调试VM #

一旦你退出所有现场tmateWindows，调试VM将终止

在重新启动作业之前将作业状态重置为原始状态。

没有更多阶段（before_install，install等等）将被执行。
