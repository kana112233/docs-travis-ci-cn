条件可用于过滤掉，并拒绝构建，阶段和作业

指定构建配置中的条件（您的.travis.yml文件）。

看到条件构建，阶段和作业

详情。

可以使用以下方法测试条件travis-conditions命令。看到

测试条件。

积分 #

使用条件进行解析和评估这个图书馆，

由接受构建请求的组件生成构建，

阶段和工作。

可以使用以下已知属性，并且可以在以下条件中使用：

  • type（当前事件类型，已知事件类型是：push，pull_request，api，cron）
  • repo（当前的存储库slugowner_name/name）
  • branch（当前分支名称;对于pull请求：基本分支名称）
  • tag（当前标签名称）
  • commit_message（当前提交消息）
  • sender（事件发件人的登录名）
  • fork（true要么false取决于存储库是否为fork）
  • head_repo（对于拉取请求：头部存储库slugowner_name/name）
  • head_branch（对于pull请求：头存储库分支名称）
  • os（操作系统）
  • language（构建语言）
  • sudo（sudo访问）
  • dist（分布）
  • group（图像组）

此外，您的构建配置中的环境变量（.travis.yml）和

存储库设置可用，可以使用匹配env(FOO)，见

下面。

请注意，这意味着条件无法访问构建环境，

他们是不用Bash评估。 Bash变量或子进程可以不

被评估。以美元字符开头的变量名和不带引号的字符串

$引发解析错误，导致构建请求被拒绝。引

字符串仍然可以以美元字符开头，所以如果你肯定需要一个字符串

以美元字符开头，您可以将其括在引号中。

规格 #

解析和评估以下表达式。

个人条款：

true
false

比较值：

1 = 1
true != false

比较函数调用属性：

env(FOO) = type

比较函数调用：

env(FOO) = env(BAR)

与正则表达式匹配：

branch =~ /^(one|two)-three$/
env(FOO) =~ /^(one|two)-three$/

列表中的属性：

branch IN (one, other)

列表中的函数调用：

repo IN (env(ONE), env(OTHER))

谓词：

tag IS present
tag IS blank


env(env(FOO))


branch = master AND env(FOO) = foo
branch = master OR env(FOO) = foo
branch = master AND env(FOO) = foo OR tag = bar
branch = master AND (env(FOO) = foo OR tag = bar)
NOT branch = master


(tag =~ ^v) AND (branch = master)
((tag =~ ^v) AND (branch = master))

ANDOR，NOTIN，IS，属性和功能

布尔运算符 #

支持以下布尔运算符：

  • AND
  • OR
  • NOT

AND结合力强于OR，和NOTAND。因此

以下表达式是相同的：

branch = master AND os = linux OR tag = bar
(branch = master AND os = linux) OR tag = bar

NOT branch = master AND os = linux
NOT (branch = master) AND os = linux

值 #

值是不带引号的字符串，不包含任何空格

或特殊字符，或单引号或双引号字符串：

"a word"
'a word'
a_word

平等和不平等 #

branch = master
sender != "my bot"
env(foo) = bar
"bar" = env("foo")

常用表达 #

这使用正则表达式匹配字符串：

branch =~ ^master$
env(foo) =~ ^bar$

如果表达式需要包含空格，或者以括号结尾，则需要

branch =~ /(master|foo)/

也可以写成master|foo）。如果你确实需要结束常规

带括号的表达式，或者如果它包含空格，则为整数

函数调用 #

有两种功能可用：

  • env
  • concat

功能env

env(FOO)

env支持在中设置的环境变量env关键（包括

env.global.travis.yml

请注意，没有功能env.global或类似的。相反所有

环境变量可通过env。

功能concat接受任意数量的参数并连接它们

成一个字符串：

concat("foo", "-", env(BAR))
# => "foo-bar"

这也可以在匹配正则表达式的上下文中使用。对于

例子env: SERVICE=some-service以下相当于

branch =~ ^srv-some-service-：

branch =~ concat(^srv-,env(SERVICE),-)

清单 #

这与值的列表（数组）匹配：

branch IN (master, dev)
env(foo) IN (bar, baz)

请注意，需要使用逗号分隔值。

应引用包含空格或特殊字符的值：

env(foo) IN ("bar baz", "buz bum")

运营商IN可以否定如下：

# these are the same
NOT branch IN (master, dev)
branch NOT IN (master, dev)

谓词 #

已知的谓词是：

present
blank
true
false

这需要存在或缺少值：

branch IS present
branch IS blank
env(foo) IS present
env(foo) IS blank

运营商IS可以否定如下：

# these are all the same
env(foo) IS NOT present
NOT env(foo) IS present
env(foo) IS blank

注意操作员IS旨在与众所周知的谓词一起使用

present和blank。它是不同样的=和表达式一样

以下做不工作：

# this does not work
branch IS "master"

# instead use =
branch = "master"

然而，IS能够用于匹配布尔谓词true和

false（在我们发现许多用户期望这样做之后，这已被包括在内

工作）：

branch IS true
branch = true # this is the same

别名 #

以下别名已到位：

  • !是别名NOT
  • &&是别名AND
  • ||是别名OR
  • ==是别名=
  • ~=是别名=~

线续（多线条件）： #

我们很惊讶地看到用户期望继续使用\上班，

就像它一样，例如，在Ruby或Python中。我们喜欢这个主意，所以我们允许

下列：

if: env(PRIOR_VERSION) IS present AND \
    env(PRIOR_VERSION) != env(RELEASE_VERSION) AND \
    branch = master AND \
    type = push

使用YAML多行字符串：

if: |
  env(PRIOR_VERSION) IS present AND \
  env(PRIOR_VERSION) != env(RELEASE_VERSION) AND \
  branch = master AND \
  type = push

例子 #

仅在主分支上构建（注意PR，这是基本分支

名称）：

branch = master

仅在标记名称与给定正则表达式匹配时构建：

tag =~ /^(v1|v2)/

仅在构建类型为或时构建push要么pull_request：

type IN (push, pull_request)

仅当分支名称不是以下几个名称之一时才构建：

branch NOT IN (master, dev)

仅当发件人登录名与给定名称匹配时才构建 - 使用引号

包含空格或特殊字符的字符串（发件人是创建构建的GitHub用户）：

sender = my_account
sender != "deploy bot"

不要在叉子上建造：

fork = false

仅在提交消息与给定正则表达式匹配时构建：

commit_message !~ /(no-deploy|wip)/

仅在Linux上构建：

os = linux
