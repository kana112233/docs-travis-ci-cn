Travis CI构建可以运行和构建Docker镜像，也可以将图像推送到

Docker存储库或其他远程存储。

要使用Docker，请将以下设置添加到您的.travis.yml：

services:

有关如何在Travis CI Enterprise上使用Docker的信息，请查看启用Docker构建。

从Dockerfile构建Docker镜像 #

而不是从中下载Docker镜像

carlad /西纳特拉您可以

直接从Dockerfile中构建它[GitHub上

知识库](https://github.com/travis-ci/docker-sinatra/blob/master/Dockerfile)。

要在当前目录中构建Dockerfile，并给它相同

carlad/sinatra标签，改变docker pull到：

docker build -t carlad/sinatra .

满满的.travis.yml看起来像这样

language: ruby

services:
  - docker

before_install:
  - docker build -t carlad/sinatra .
  - docker run -d -p 127.0.0.1:80:4567 carlad/sinatra /bin/sh -c "cd /root/sinatra; bundle exec foreman start;"
  - docker ps -a
  - docker run carlad/sinatra /bin/sh -c "cd /root/sinatra; bundle exec rake test"

script:
  - bundle exec rake test

{：data-file =“。travis.yml”}

将Docker镜像推送到注册表 #

要将映像推送到Docker注册表，必须先通过身份验证docker login。用于登录的电子邮件，用户名和密码应存储在

存储库设置环境变量，可以通过设置

存储库设置网页或通过Travis CLI本地，例如：

travis env set DOCKER_USERNAME myusername
travis env set DOCKER_PASSWORD secretsecret

务必加密环境变量

使用travis gem。

你的内心.travis.yml在尝试之前docker push或者也许之前

docker pull私人形象，例如：

echo "$DOCKER_PASSWORD" | docker login -u "$DOCKER_USERNAME" --password-stdin

基于分支的注册表推送 #

要将存储库的特定分支推送到远程注册表，

使用你的自定义部署部分.travis.yml：

deploy:
  provider: script
  script: bash docker_push
  on:
    branch: master

{：data-file =“。travis.yml”}

哪里docker_push是您的存储库中的脚本，包含：

#!/bin/bash
echo "$DOCKER_PASSWORD" | docker login -u "$DOCKER_USERNAME" --password-stdin
docker push USER/REPO

{：data-file =“docker_push”}

私人注册登录 #

推送到私有注册表时，请务必在中指定主机名

docker login命令，例如：

echo "$DOCKER_PASSWORD" | docker login -u "$DOCKER_USERNAME" --password-stdin registry.example.com

使用Docker Compose #

该Docker撰写工具也是安装在Docker启用的环境中。

如果需要，您可以轻松替换此预安装版本docker-compose

通过添加以下内容before_install走向你的.travis.yml：

env:
  - DOCKER_COMPOSE_VERSION=1.4.2

before_install:
  - sudo rm /usr/local/bin/docker-compose
  - curl -L https://github.com/docker/compose/releases/download/${DOCKER_COMPOSE_VERSION}/docker-compose-`uname -s`-`uname -m` > docker-compose
  - chmod +x docker-compose
  - sudo mv docker-compose /usr/local/bin

{：data-file =“。travis.yml”}

安装较新的Docker版本 #

您可以手动升级到最新版本并使用任何新的Docker功能

更新它before_install你的一步.travis.yml：

从download.docker.com更新

before_install:
  - curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo apt-key add -
  - sudo add-apt-repository "deb [arch=amd64] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable"
  - sudo apt-get update
  - sudo apt-get -y -o Dpkg::Options::="--force-confnew" install docker-ce

{：data-file =“。travis.yml”}

或者，您可以使用addons代替before_install通过更新apt以及：

addons:
  apt:
    packages:
      - docker-ce

{：data-file =“。travis.yml”}

检查您正在运行的Docker版本docker --version
