Travis CI Enterprise SSL Certificate Management

  • 使用自签名证书
  • 如何在安装后添加或更新证书

此页面包含有关SSL证书及其在Travis CI Enterprise中的使用的信息。

默认情况下，Travis CI Enterprise会为内部平台组件RabbitMQ和Github.com/Github Enterprise之间的所有流量验证SSL。 SSL验证有助于确保安全地传输流量。它是强力推荐使用来自受信任机构的有效证书配置安装。

如果您对特定部分有疑问或遇到未涵盖的问题，请通过以下方式与我们联系enterprise@travis-ci.com寻求帮助。

使用自签名证书 #

我们强烈建议您在配置安装时使用来自受信任的证书颁发机构的有效证书。也就是说，我们认识到使用自签名证书可能会对您的组织更有意义。

在所有使用自签名证书的情况下，您必须禁用SSL证书验证在你的设置中。这可以在安装完成后完成。

安装完成后，您可以随时上传证书管理仪表板。

使用试用许可证 #

如果您使用试用许可证来评估Travis CI Enterprise，则使用可信证书可能没有意义。对于此方案，我们提供了使用由Travis CI Enterprise自身生成的自签名证书的功能。

为此，您可以在提示安装期间提供自定义SSL证书时选择“使用自签名证书”选项。

具有内部管理的证书颁发机构的组织 #

有些组织管理自己的内部证书颁发机构。内部机构可能或可能使用由a提供的根证书受信任的root权限。

没有来自受信任的证书颁发机构的根证书 #

如果您的内部证书颁发机构未使用受信任的证书颁发机构提供的根证书，那么大多数工具和浏览器都会将生成的任何证书视为“不受信任”，与自签名证书相同。

内部证书颁发机构创建的证书可以在安装期间以与任何其他证书相同的方式上载。

请注意，您还需要确保遵循有关禁用SSL验证的说明以下为使用Travis CI Enterprise。

使用来自受信任的证书颁发机构的根证书 #

如果您组织的内部证书颁发机构确实使用从受信任的证书颁发机构购买的根证书，则上述内容将不适用。您无需禁用SSL验证，可以像直接从受信任的证书颁发机构使用证书一样继续操作。

在某些情况下，可能需要在设置中包含中间证书链，以便SSL验证正常工作。原因可能因内部设置和根证书提供商而异。请参阅下面的部分如何设置中间证书链作为指示。

禁用SSL验证 #

SSL验证有助于确保Travis CI Enterprise组件之间的所有流量都受到保护。禁用SSL验证会产生严重影响！

要在Travis CI Enterprise中禁用SSL验证：

  1. 登录您的信息中心https://<your-travis-ci-enterprise-domain>:8800。
  2. 单击最顶层菜单中的“设置”。
  3. 单击左侧菜单中的“高级设置”（或向下滚动）。
  4. 选中“禁用SSL证书验证”复选框。
  5. 点击页面底部的“保存”。请注意，这将需要重新启动Travis CI Enterprise才能生效并可能导致短暂停机。

这将删除Travis CI组件之间内部流量的SSL验证检查。这也将导致在Github上创建的所有存储库挂钩关闭来自Travis CI的流量的SSL验证。请注意，此设置仅适用于创建的挂钩后设置已启用。

如何在安装后添加或更新证书 #

您可以在安装后随时通过管理仪表板添加或更新证书。

要上传您的证书：

  1. 登录您的信息中心https://<your-travis-ci-enterprise-domain>:8800。
  2. 单击顶部菜单最右侧的齿轮图标，然后从下拉列表中选择“控制台设置”。
  3. 单击左侧菜单中的“TLS Key＆Cert”（或向下滚动）。
  4. 根据证书文件的位置选择适当的选项。
  5. 输入路径或选择要上载的文件。
  6. 点击页面底部的“保存”。请注意，这将需要重新启动Travis CI Enterprise才能生效并可能导致短暂停机。

需要有效的x509证书和私钥文件。证书和密钥必须采用PEM格式。密钥必须是未加密的。

##

在某些情况下，可能需要提供为您所需的证书。原因可能因内部设置和根证书提供商而异。

如果您已上载有效证书但仍看到与SSL验证错误相关的意外问题，则可以在设置中设置中间证书链。

  1. https://<your-travis-ci-enterprise-domain>:8800。
  3. 单击左侧菜单中的“高级设置”（或向下滚动）。

##

你需要什么：

  • 电子邮件地址（Let’s Encrypt将发送有关紧急续订和安全问题的通知）。
  • 用于安装的域名。

以下步骤需要Travis CI Enterprise实例的停机时间。因此，我们建议您在维护时段执行这些步骤。

安装certbot #

我们将使用certbot从Let’s Encrypt获取SSL证书。要安装certbot：

  2. 添加certbot个人包存档：
sudo add-apt-repository ppa:certbot/certbot


sudo apt-get update
  1. 安装所需的certbot依赖项：
sudo apt-get install software-properties-common
  1. 安装certbot包：
sudo apt-get install certbot

生成一个新的Let的加密证书 #

certbot工具提供了多种获取证书的方法。我们将选择“临时网络服务器”选项，因为它不需要任何其他设置或配置。唯一的先决条件是必须停止Travis CI Enterprise容器，以便certbot Web服务器可以正确绑定到端口443。

在此过程中，您的Travis CI Enterprise实例将不可用。

要生成证书：

  1. 打开与平台机器的ssh连接。
  2. 停止您的Travis CI企业：
replicatedctl app stop
  1. 启动交互式证书流程：
sudo certbot certonly
  1. 在提示选择选项1Spin up a temporary webserver：
How would you like to authenticate with the ACME CA?
-------------------------------------------------------------------------------
1: Spin up a temporary webserver (standalone)
2: Place files in webroot directory (webroot)
-------------------------------------------------------------------------------
Select the appropriate number [1-2] then [enter] (press 'c' to cancel): 1
  1. 填写您的电子邮件地址：
Enter email address (used for urgent renewal and security notices) (Enter 'c' to
cancel): ops@example.com
  1. 接受服务条款：
-------------------------------------------------------------------------------
Please read the Terms of Service at
https://letsencrypt.org/documents/LE-SA-v1.1.1-August-1-2016.pdf. You must agree
in order to register with the ACME server at
https://acme-v01.api.letsencrypt.org/directory
-------------------------------------------------------------------------------
(A)gree/(C)ancel: A
  1. 确定您是否要与EFF共享您的电子邮件地址（此决定对您的证书或Travis CI Enterprise实例没有影响）：
-------------------------------------------------------------------------------
Would you be willing to share your email address with the Electronic Frontier
Foundation, a founding partner of the Let's Encrypt project and the non-profit
organization that develops Certbot? We'd like to send you email about EFF and
our work to encrypt the web, protect its users and defend digital rights.
-------------------------------------------------------------------------------
(Y)es/(N)o: N
  1. 提供您的域名：
Please enter in your domain name(s) (comma and/or space separated)  (Enter 'c'
to cancel): <your-travis-ci-enterprise-domain>
  1. 成功完成后，您应该看到类似于以下内容的消息：
IMPORTANT NOTES:
   Congratulations! Your certificate and chain have been saved at:
   /etc/letsencrypt/live/<your-travis-ci-enterprise-domain>/fullchain.pem
   Your key file has been saved at:
   /etc/letsencrypt/live/<your-travis-ci-enterprise-domain>/privkey.pem
   Your cert will expire on 2018-02-07. To obtain a new or tweaked
   version of this certificate in the future, simply run certbot
   again. To non-interactively renew *all* of your certificates, run
   "certbot renew"
  1. 重新启动Travis CI Enterprise实例：
replicatedctl app start

您的证书现在已生成并保存在Travis CI Enterprise平台计算机上，但您必须采取其他步骤来配置Travis CI Enterprise实例以使用新证书。请看看使用Let的加密证书部分说明。

使用Let的加密证书 #

在此过程中，您的Travis CI Enterprise实例将不可用。

要在Travis CI Enterprise实例中使用生成的证书，请执行以下操作：

  1. 登录您的信息中心https://<your-travis-ci-enterprise-domain>:8800。
  2. 在顶部菜单的最右侧，单击齿轮图标，然后从下拉列表中选择“控制台设置”。
  3. 单击左侧菜单中的“TLS Key＆Cert”（或向下滚动）。
  4. 选择“服务器路径”选项。
  5. 输入上面certbot输出中提供的证书路径。例：
- SSL Private Key Filename: `/etc/letsencrypt/live/<your-travis-ci-enterprise-domain>/privkey.pem`
- SSL Certificate Filename: `/etc/letsencrypt/live/<your-travis-ci-enterprise-domain>/fullchain.pem`
  1. 点击页面底部的“保存”。
  2. 打开与平台机器的ssh连接。
  3. 停止您的Travis CI Enterprise实例：
replicatedctl app stop
  1. 重新启动Travis CI Enterprise实例：
replicatedctl app start

续订证书 #

我们的加密证书是短暂的90天后过期。因此，您需要定期更新它们。这可以使用certbot手动完成。

在此过程中，您的Travis CI Enterprise实例将不可用。

要将证书手动续订另外90天：

  2. 停止您的Travis CI Enterprise实例：
replicatedctl app stop
  1. 执行证书续订：
sudo certbot renew
  1. 重新启动Travis CI Enterprise实例：
replicatedctl app start

使用cron作业自动续订Let的加密SSL证书 #

您可以通过使用cron作业自动执行certbot续订来避免手动续订：

  1. 创建/home/ubuntu/renew-certs.sh包含以下脚本： ``` #!/bin/bash

set -u

function usage() { echo echo “Usage: $(basename $0)” echo echo “Simple script to renew certs, should be used via a cron. Assumes that certs are already set up.” echo echo “See https://docs.travis-ci.com/user/enterprise/platform-tips/#use-a-lets-encrypt-ssl-certificate for initial setup info.” echo

exit 1 }

if [[ $# -ne 0 ]]; then usage fi

replicatedctl app stop sudo certbot renew replicatedctl app start


{：data-file =“/ home / ubuntu / renew-certs.sh”}
2. 使脚本可执行：

$ chmod +x /home/ubuntu/renew-certs.sh


3. 通过编辑创建一个cronjob`/etc/crontab`并附加以下内容：

Renews certs at 2am on the 1st of February, May, August, and November. #

Please change the configuration that applies to the certificate you are creating. #

0 2 1 2,5,8,11 * /home/ubuntu/renew-certs.sh

```

{：data-file =“/ etc / crontab”}

更新证书时，此过程将引入少量停机时间。

我们建议您在每次续订之前与您的用户进行通信，以便他们知道在续订证书之前，他们的构建将暂时停止。
