构建超时 #

挂起测试套件或构建脚本是很常见的。

Travis CI对每个作业都有特定的时间限制，并将在以下情况下停止构建并向构建日志添加错误消息：

  • 当作业在10分钟内没有产生日志输出时。
  • 当公共存储库上的作业花费超过50分钟时。
  • 当私有存储库上的作业超过120分钟时。

构建可能会挂起的一些常见原因：

  • 并发问题（死锁，活锁等）
  • 安装本机扩展，需要很长时间才能编译

限制并发作业 #

您可以在设置窗格中设置最大并发作业数

每个存储库。

$ travis settings maximum_number_of_builds --set 1

仅构建最新提交 #

如果您只对在每个分支上构建最新提交感兴趣，可以使用此新功能自动取消队列中的旧版本尚未运行。现有版本将被允许完成。

该自动取消设置在里面设置每个存储库的选项卡，您可以单独启用它：

  • 自动取消分支构建 - 取消分支中的排队构建并显示在建立历史您的存储库的选项卡。
  • 自动取消拉取请求构建 - 取消拉取请求的排队构建（更改/功能分支与其目标的未来合并结果）并显示在请求您的存储库的选项卡。

Git克隆深度 #

Travis CI可以将存储库克隆到最大深度为50次提交，这在您执行git操作时才真正有用。

请注意，如果您使用深度为1并且具有作业队列，则在您推送新提交时，Travis CI将不会构建队列中的提交。

你可以设置克隆深度在.travis.yml：

git:
  depth: 3

{：data-file =“。travis.yml”}

你也可以删除--depth标志完全与：

git:
  depth: false

{：data-file =“。travis.yml”}

存储库上的某些操作（例如常见的自动代码审查脚本（例如，Pronto for Ruby））可能由于git克隆深度有限而无法访问存储库中的所有对象而失败。删除深度标记或运行git fetch --unshallow可能会解决这个问题。

Git Clone Quiet #

Travis CI克隆没有安静标志的存储库（-q） 默认情况下。如果您尝试避免日志文件大小限制，或者即使您不需要包含它，也可以启用quiet标志。

你可以启用安静的旗帜在.travis.yml：

git:
  quiet: true

{：data-file =“。travis.yml”}

Git子模块 #

Travis CI默认克隆Git子模块，以避免此设置：

git:
  submodules: false

{：data-file =“。travis.yml”}

Git Sparse Checkout #

Travis CI支持git的稀疏结账

能力。

要稀疏地克隆您的存储库，请添加：

git:
  sparse_checkout: skip-worktree-map-file

{：data-file =“。travis.yml”}

哪里skip-worktree-map-file是当前存储库中现有文件的路径，其中包含您要放入的数据$GIT_DIR/info/sparse-checkout的文件Git文档中描述的格式。

禁用git clone #

在某些工作流程中，例如建立阶段，跳过自动可能是有益的git clone步。

你可以通过添加：

git:
  clone: false

请注意，如果您使用此选项，则TRAVIS_COMMIT_MESSAGE环境变量将不会被定义。

建立特定的分支机构 #

特拉维斯CI使用.travis.yml来自包含触发构建的Git提交的分支的文件。使用安全列表包含分支，或使用阻止列表将其排除。

请注意，您还需要考虑自动拉取请求构建在决定对某些分支进行安全列表或阻止列表时。

安全或阻止分支机构 #

使用安全列表指定要构建的分支，或者不希望构建的阻止列表分支：

# blocklist
branches:
  except:
  - legacy
  - experimental

# safelist
branches:
  only:
  - master
  - stable

{：data-file =“。travis.yml”}

请注意，安全列表还可以防止构建标记的提交。如果您始终以格式标记构建v1.3你可以用它们来安全常用表达， 例如/^v\d+\.\d+(\.\d+)?(-\S*)?$/。

如果同时使用安全列表和阻止列表，则安全列表优先。默认情况下gh-pages除非您将分支添加到安全列表中，否则不会构建分支。

建立所有分支：

branches:
  only:
  - gh-pages
  - /.*/

{：data-file =“。travis.yml”}

请注意，由于历史原因.travis.yml需要出席在所有活跃的分支上你的项目。

使用正则表达式 #

您可以使用正则表达式来安全列表或阻止列表分支：

branches:
  only:
  - master
  - /^deploy-.*$/

{：data-file =“。travis.yml”}

任何名字包围着/在分支列表中被视为正则表达式，可以包含任何支持的量词，锚点或字符类Ruby正则表达式。

最后一个之后指定的选项/（例如。，i对于不区分大小写的匹配）不受支持，但可以内联给出。例如，/^(?i:deploy)-.*$/火柴Deploy-2014-06-01和别的

以…开头的分支和标签deploy-在任何情况下的组合。

跳过构建 #

如果您不想出于任何原因为特定提交运行构建，则可以指示Travis CI

通过提交消息中的命令跳过构建此提交。

该命令应为以下形式之一：

[跳过]要么

[skip ]哪里``或者是`ci`，`travis`，`travis ci`，`travis-ci`， 要么`travisci`。

例如，

[skip travis]更新自述文件请注意，如果将多个提交一起推送，则skip命令仅在HEAD提交的提交消息中存在时才有效。

构建矩阵 #

您还可以定义构建矩阵的排除项：

matrix:
  exclude:
  - rvm: 1.9.3
    gemfile: gemfiles/Gemfile.rails-2.3.x
    env: ISOLATED=true
  - rvm: jruby
    gemfile: gemfiles/Gemfile.rails-2.3.x
    env: ISOLATED=true

{：data-file =“。travis.yml”}

目前，所有构建矩阵都限制为最大值200个工作私有和公共存储库。如果您使用的是开源计划，请记住，Travis CI向社区免费提供此服务。所以请只指定你的矩阵实际需要。

在矩阵中命名作业 #

您可以为矩阵中的特定作业定义名称。我们建议使用独特的工作名称

不强制执行（虽然将来可能会改变）。职位定义在matrix.include

部分可以给出一个工作名称，如下所示：

language: python
matrix:
  include:
  - name: "3.5 Unit Test"
    python: "3.5"
    env: TEST_SUITE=suite_3_5_unit
  - name: "3.5 Integration Tests"
    python: "3.5"
    env: TEST_SUITE=suite_3_5_integration
  - name: "pypy Unit Tests"
    python: "pypy"
    env: TEST_SUITE=suite_pypy_unit
script: ./test.py $TEST_SUITE

{：data-file =“。travis.yml”}

通过矩阵扩展生成的作业不能赋予名称属性。

快速整理 #

要尽快将构建标记为已完成，请添加fast_finish: truematrix你的部分.travis.yml像这样：

matrix:
  fast_finish: true

现在，一旦所有必需的工作完成，将根据这些结果确定构建结果，而其余部分allow_failures工作继续运行。
