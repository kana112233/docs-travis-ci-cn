2018年5月2日，Travis CI宣布将开源项目加入travis-ci.com上的私人项目！

本文档介绍了如何迁移存储库，并回答了有关迁移的一些常见问题。

嗨，您好！如果您想成为测试版测试人员，请发送电子邮件给我们主题为“开源迁移Beta测试”的主题为support@travis-ci.com - 我们会在那里向您发送更多详细信息！

什么信息将被转移到travis-ci.com？ #

迁移存储库时，以下信息将传输到travis-ci.com：

  • 为存储库配置的Cron作业
  • 上次构建状态 - travis-ci.com中的构建徽章将反映此状态，直到触发新构建
  • 下一个版本号

哪些信息不会转移到travis-ci.com？ #

迁移存储库时，以下信息不会传输到travis-ci.com。

  • 以前的构建历史 - 从迁移到travis-ci.com之前的构建将保留在travis-ci.org上，直到完成构建历史迁移（2019年第二季度的某个时间）
  • 缓存 - 它们将在travis-ci.com上的第一次构建时自动重新创建

我是否需要进行任何其他更改？ #

  • 建立徽章：确保更新URL以反映travis-ci.com中的新存储库设置。将来，我们会自动将travis-ci.org徽章网址重定向到相应的travis-ci.com网址。
  • GitHub中的“受保护的分支”功能在合并Pull Request之前要求传递Travis CI构建：确保编辑所需的状态检查以便现在使用Travis CI - Pull Request和/或Travis CI - Branch在GitHub上进行必要的状态检查
  • Travis CLI：迁移后一定要添加--com国旗或将它设为您的默认端点通过运行：travis endpoint --com --set-default
  • Travis CI API：编辑您的请求以使用新端点：api.travis-ci.comapi.travis-ci.org。

我的travis-ci.org存储库会发生什么？ #

**。

使用此只读模式：

  • 所有以前的构建历史记录都可以在travis-ci.org上访问
  • travis-ci.org中将不提供“触发器构建”功能
  • 无法重新启动在travis-ci.org中运行的先前作业
  • 存储库的“设置”页面将变为非活动状态 - 转移的存储库设置现在将在travis-ci.com中提供。

将来，当迁移所有以前的构建历史记录时，我们将提供从travis-ci.org到travis-ci.com的所有API / Web请求的重定向。

迁移过程需要多长时间？ #

存储库的迁移过程不应超过几秒钟。

如果有人在迁移时将提交推送到我的存储库会发生什么？ #

我们将这些构建请求排入队列，一旦迁移完成，将在travis-ci.com中创建构建。

迁移存储库 #

GitHub Apps初始设置 #

  1. 登录到https://travis-ci.comhttps://travis-ci.com/profile

  1. 激活Travis CI GitHub应用程序时，授予您要在travis-ci.com中构建的存储库（公共和私有）的访问权限。保存更改。

迁移步骤 #

  1. 一旦您通过GitHub应用程序授予对您要构建和传输的存储库的访问权限，就可以使用新的“迁移”标签将显示在您的帐户页面中，其中包含要迁移的存储库列表：

  1. 选择您要迁移的存储库，然后单击“迁移选定的存储库”，系统将要求您进行最终确认 - 请记住此时不会迁移您的构建历史记录（请参阅：哪些信息不会转移到travis-ci.com？）。
  2. 确认迁移。存储库名称旁边的图标将显示迁移状态（“处理”或“已迁移”）：

  1. 而已！您的开源存储库现在可以在travis-ci.com上构建了！

通过API迁移存储库 #

如果您希望自动化迁移过程，还可以通过直接向其发送请求来迁移存储库/repo/:id/migrate要么/repo/:slug/migrateTravis CI API的端点：

  • 使用存储库slug：
 curl -s -X POST \
   -H "Content-Type: application/json" \
   -H "Accept: application/json" \
   -H "Travis-API-Version: 3" \
   -H "Authorization: token {API_TOKEN}" \
   --data '' \
   https://api.travis-ci.com/repo/{REPO_OWNER}%2F{REPO_NAME}/migrate
  • 使用存储库ID：
 curl -s -X POST \
   -H "Content-Type: application/json" \
   -H "Accept: application/json" \
   -H "Travis-API-Version: 3" \
   -H "Authorization: token {API_TOKEN}" \
   --data '' \
   https://api.travis-ci.com/repo/{REPO_ID}/migrate

与迁移的存储库交互 #

Travis CI现在将开始在travis-ci.com中接收迁移的开源存储库的GitHub事件。任何新的构建和请求都将开始出现在travis-ci.com网站上。

travis-ci.com中迁移的存储库的项目页面将开始显示以下消息：

然后，当返回到travis-ci.org时，迁移的存储库仍将出现在您的存储库列表中，并带有直接链接以访问travis-ci.com中的项目：

由于travis-ci.org中的存储库现在处于只读模式，因此设置页面还将链接到travis-ci.com中的相应设置页面：
