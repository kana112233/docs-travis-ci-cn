上2018年5月2日Travis CI宣布开源项目将加入私人项目travis-ci.com！

这意味着您可以从一个域管理公共和私有存储库，为开源或私有项目分配额外的并发性，并在项目类型之间获得更一致的体验。

我们正在使用GitHub应用程序实现这些更改，它提供了与GitHub的更紧密集成以及对存储库和订阅所有权和权限的更精细控制。构建的新的或迁移的存储库travis-ci.com将可以访问基于GitHub Apps的集成提供的功能。

您能期待什么 #

这是Travis CI的重大发展，将意味着在转换期间以及之后对存储库和用户管理的发生方式进行一些更改。新的开源用户帐户和存储库将能够使用travis-ci.com马上，现有的用户帐户和存储库将随着时间的推移迁移。

GitHub应用程序集成的功能 #

与我们的Legacy Webhooks集成相比，基于GitHub-Apps的集成带来了许多好处：

  • 在Travis CI上激活时，我们将不再向存储库添加部署密钥
  • 项目将使用寿命较短的OAuth令牌进行克隆
  • 我们的GitHub应用程序所需的权限远低于我们的OAuth应用程序
  • 组织存储库的所有权始终与拥有该项目的组织相关联

新用户帐户 #

请注册travis-ci.com，无论您是否计划测试开源或私有项目。欢迎登机！

现有用户帐户 #

目前的用户将保留他们的travis-ci.com和travis-ci.org首先是帐户。我们推荐用户travis-ci.org激活任何新的存储库上travis-ci.com。如果您还没有注册travis-ci.com如果您想激活新的存储库，请先在那里注册。

新的存储库 #

应该激活新的存储库travis-ci.com对于开源和私有存储库。您还可以选择立即订阅开源项目以获得更多并发性，而不是首先需要联系支持。

travis-ci.org上的现有开源存储库 #

开源项目及其构建历史将继续运行travis-ci.org此时。

但是，您可以参与封闭式Beta版测试，以开始将开源资源库迁移到travis-ci.com。

  1. 要迁移任何公共存储库，必须首先激活它travis-ci.com使用GitHub应用程序。
  2. 发送电子邮件给我们主题为“开源迁移Beta测试”的主题为support@travis-ci.com
  3. 查看beta测试包含的内容travis-ci.com迁移指南的开源。

现有的私有存储库travis-ci.com #

您的私人存储库不会发生任何变化。私人项目将继续运行travis-ci.com像之前一样。

但是，我们建议您通过GitHub应用程序激活您的私人存储库，以开始充分利用这种新的集成。

继续阅读私有存储库的GitHub应用程序指南的旧服务。

联系支持 #

如果您有任何疑问，请发送电子邮件至联系support@travis-ci.com。我们期待着帮助！
