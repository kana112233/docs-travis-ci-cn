Migrating 50 or more private Repositories to GitHub Apps Guide

如果您的组织有超过50个活动存储库要迁移到我们新的GitHub Apps集成，您可以使用travis_migrate_to_apps宝石在一次扫描中迁移它们，而不是在GitHub UI中逐个手动添加它们。

请注意，您还可以在GitHub上选择“激活所有存储库”以选择所有当前和未来的存储库，以便您可以在Travis CI中通过添加.travis.yml文件。

以下步骤描述了如何通过GitHub应用程序首先授权您的帐户，获取所需的身份验证令牌并使用travis_migrate_to_apps宝石通过GitHub应用程序激活您的私人存储库：

1.安装宝石 #

gem install travis_migrate_to_apps

2.使用repo范围生成GitHub个人访问令牌 #

您可以生成新的GitHub令牌这里。

选择您喜欢的名称，并确保选择整体repo范围如下图所示：

然后单击底部的“生成令牌”按钮以生成令牌。

然后，您将返回GitHub令牌页面：

注意复制新生成的令牌并保存以供以后使用。

注意：如果要迁移组织，则必须生成令牌GitHub组织的所有者。

3.使用1个repo激活Travis CI GitHub Apps集成 #

转到travis-ci.com上的存储库页面：https://travis-ci.com/account/repositories

单击下面突出显示的“激活GitHub应用程序集成”按钮：

您将转到Travis CI应用程序的GitHub应用程序页面：

选择至少一个存储库，然后单击“批准和安装”按钮。

然后，您将被重定向到您在Travis CI上的个人资料页面并且新添加的存储库应显示在“GitHub Apps Integration”下：

4.获取Travis CI API令牌 #

以下是获取此令牌的两种方法：

  1. 在您的个人资料页面上：https：//travis-ci.com/profile

  1. 通过运行：travis token --com

5.运行gem #

travis_migrate_to_apps [owner_name] [travis_access_token] [github_access_token]

哪里

  • [owner_name]是要迁移的存储库所在的GitHub帐户（用户或组织）
  • [travis_access_token]是上面步骤＃4中获得的Travis CI令牌
  • [github_access_token]是上面步骤＃2中获得的GitHub令牌

###

你现在可以刷新了并且您新迁移的存储库应显示在“GitHub Apps Integration”部分下。
