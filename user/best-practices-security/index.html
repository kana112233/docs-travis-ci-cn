Travis CI用于保护您的数据的步骤 #

Travis CI模糊了UI中显示的安全环境变量和令牌。我们的有关加密密钥的文档

[secure]代替。

请确保您的秘密永远不会与存储库或分支名称或任何其他可猜测的字符串相关。理想情况下使用密码生成工具，如mkpasswd而不是自己选择一个秘密。

测试版Windows支持不会混淆泄露到构建日志中的安全环境变量。请继续阅读下一节，关于如何避免泄漏秘密来构建日志

关于如何避免泄漏秘密以构建日志的建议 #

尽管我们付出了最大努力，但仍有许多方法可以意外地暴露安全信息。这些因您使用的工具和启用的设置而异。需要注意的一些事项是：

  • 将命令复制到标准输出的设置，例如set -x要么set -v在你的bash脚本中
  • 通过运行显示环境变量env要么printenv
  • 例如，在代码中打印机密echo "$SECRET_KEY"
  • 使用在错误输出上打印机密的工具，例如php -i
  • git命令就像git fetch要么git push可能会暴露令牌或其他安全环境变量
  • 字符串转义中的错误
  • 增加命令详细程度的设置
  • 可能在操作时暴露机密的测试工具或插件

防止命令显示任何输出是避免意外显示任何安全信息的一种方法。如果存在使用安全信息的特定命令，则可以将其输出重定向到/dev/null确保它不会意外地发布任何内容，如以下示例所示：

git push url-with-secret >/dev/null 2>&1

如果您认为自己可能暴露了安全信息 #

作为第一步，可以通过单击删除包含任何安全信息的日志删除日志Travis CI的构建日志页面上的按钮。

如果您发现某个构建日志中存在泄漏，则必须撤消泄漏的令牌或环境变量，并更新导致泄漏的任何构建脚本或命令。

删除日志的替代方法 #

您可以使用。来代替手动删除构建日志Travis CI CLI或者API。

请注意，如果您仍在使用travis-ci.org你需要使用开源API代替。

定期旋转令牌和秘密 #

定期旋转你的代币和秘密。 GitHub OAuth令牌可以在你的开发者设置在GitHub网站上。请定期轮换其他第三方服务的凭据。

更多信息 #
