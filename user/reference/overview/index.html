我的环境运行在哪个基础架构上？ #

通常，从中了解虚拟化环境特征上面的表足够了。

但是，如果您确实需要更多细节，那么您有以下两个问题之一：

  • 你想看看什么基础设施完成构建跑了
  • 你想确定什么基础设施特定.travis.yml组态将继续。

完成构建 #

要查看完成的构建运行的基础架构，请查看主机名在构建日志的顶部：

如果它包含：

  • gce→构建在Google Compute Engine上的虚拟机中运行。
  • wjb→构建在macOS上运行。
  • 1803-containers→构建在Windows上运行。

对于特定的.travis.yml配置 #

  • 我们的默认基础设施是Ubuntu Linux（os: linux）在Google Compute Engine上运行的虚拟机。您可以使用指定哪个版本的Ubuntudist键。
  • 运用os: osx，使用设置Xcode的版本osx_image:，或使用macOS特定语言，如language: objective-c将您的构建路由到macOS基础架构。
  • 运用os: windows将构建路由到Windows基础结构。
