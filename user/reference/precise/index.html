本指南涵盖的内容 #

CI环境（通常称为“CI环境”）。

概观 #

Travis CI在提供vanilla构建的独立虚拟机中运行构建

这样做的好处是，构建之间不会出现任何状态，从而提供干净的状态

陈述并确保您的测试在从头开始构建的环境中运行。

构建可以访问各种用于数据存储和消息传递的服务

可以安装他们运行所需的任何东西。

联网 #

运行测试的旧环境中的虚拟机已启用IPv6。它们没有任何外部IPv4地址，但完全能够与任何外部IPv4服务通信。

基于容器，macOS和GCE（包括Precise和Trusty）构建当前没有IPv6连接。

IPv6堆栈尤其会对Java服务产生一些影响，可能需要设置标志java.net.preferIPv4Stack如果服务显示无法启动或无法通过网络访问的问题，则强制JVM诉诸IPv4堆栈：-Djava.net.preferIPv4Stack=true。

大多数服务在与本地主机通话时都能正常工作localhost要么127.0.0.1

火狐 #

目前，所有虚拟环境都安装了最新版本的Firefox

Linux环境为31.0，macOS为25.0。

如果您需要特定版本的Firefox，请使用Firefox插件进行安装

它在期间before_install构建阶段。

例如，要安装17.0版，请将以下内容添加到您的

.travis.yml文件：

addons:
  firefox: "17.0"

{：data-file =“。travis.yml”}

请注意，插件仅适用于64位Linux环境。

集团成员 #

执行构建的用户（$USER

如果您的项目需要额外的成员资格来运行构建，请按照下列步骤操作：

  1. 设置环境。这可以在构建脚本执行之前的构建生命周期中的任何时间完成。
    1. 设置和导出环境变量。
    2. 加$USERsudo usermod -a -G SECONDARY_GROUP_1,SECONDARY_GROUP_2 $USER

您可以使用修改用户的主要组-g

  1. script看起来像是这样的： ``` script: sudo -E su $USER -c ‘COMMAND1; COMMAND2; COMMAND3’


这会将环境变量传递给a`bash`运行的过程`$USER`

同时保留定义的环境变量并属于上面给出的二级组`usermod`

### Maven版本

股票Apache Maven 3.2.x，配置使用[中央](http://search.maven.org/)和[Sonatype的](https://oss.sonatype.org/)镜子。

### 

Travis CI拥有独立（“uberjar”）Leiningen 1.7.x at`/usr/local/bin/lein1`和Leiningen 2.4.x at`/usr/local/bin/lein2`。

默认值为2.4.x;`/usr/local/bin/lein`是一个象征性的链接`/usr/local/bin/lein2`。

### SBT版本

由于功能非常强大，Travis CI可能提供任何版本的Simple Build Tool（sbt或SBT）[SBT-演员](https://github.com/paulp/sbt-extras)替代。为了减少构建时间，已预先安装了流行版本的sbt（例如0.13.5或0.12.4），但是`sbt`命令能够动态检测和安装Scala项目所需的sbt版本。

见[默认的sbt选项](https://github.com/travis-ci/travis-cookbooks/blob/precise-stable/ci_environment/sbt-extras/templates/default/sbtopts.erb)

有关使用sbt构建项目的具体细节。
