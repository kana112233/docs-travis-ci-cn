##

本指南概述了Xenial环境中可用的软件包，工具和设置。

使用Xenial #

要将您的构建路由到Ubuntu Xenial 16.04，请将以下内容添加到您的.travis.yml

dist: xenial

{：data-file =“。travis.yml”}

请注意，Xenial可在我们托管的完全虚拟化版本上使用

至enterprise@travis-ci.com了解如何使用Xenial Docker镜像。

与Trusty图像的差异 #

Xenial包括以下更改和改进：

删除了第三方apt-repositories #

虽然在Xenial映像配置期间使用了第三方apt-repositories，但它们都从Xenial构建映像中删除。这有两个好处; a）降低无关干扰的风险，以及b）更快的apt-get更新。

要指定第三方apt-repository，您可以使用apt插件添加源代码并指定包。例如：

dist: xenial
addons:
  apt:
    sources:
      - ppa:chris-lea/redis-server
    packages:
    - redis-tools
    - redis-server

{：data-file =“。travis.yml”}

如果您在构建中依赖这些存储库，则可以使用以下内容source

包裹 来源
：——————— ：————————- —-
ppa:couchdb/stable
docker-xenial
google-chrome-stable google-chrome
git-lfs github-git-lfs-xenial
git-ppa ppa:git-core/ppa
haskell ppa:hvr/ghc
mongodb mongodb-4.0-xenial
授粉 ppa:pollinate/ppa
ppa:chris-lea/redis-server

{：style =“width：80％”}

默认情况下禁用服务 #

在我们的Xenial基础架构上，为了加快启动时间并提高性能，我们禁用了所有服务，包括默认情况下在Trusty上启动的服务。

将您默认启动的任何服务添加到您的.travis.yml：

services:
  - mysql
  - redis

{：data-file =“。travis.yml”}

所有Xenial图像共有的环境 #

所有版本中都包含以下版本的Docker，版本控制软件和编译器，以及每种语言部分中更详细描述的更多语言特定软件。

编译器和构建工具链 #

  • clang和llvm 7
  • cmake 3.12.4
  • gcc 5.4.0
  • ccache 3.2.4
  • shellcheck 0.6.0
  • shfmt 2.6.3

搬运工人 #

  • 已安装Docker 18.06.0-ce
  • docker-compose 1.23.1。

Ruby支持 #

  • 预装红宝石：2.4.52.5.3
  • 默认的ruby是2.5.3p105。
  • 其他ruby版本可以在构建期间安装。

Python支持 #

  • 支持的Python版本：2.7，3.4或更高。
  • 预安装的Python版本：2.7.15，3.6.7，和3.7.1。
  • 蟒蛇2.7.15将在未明确设置语言版本时使用。

如果你有关于PyPy的错误pypy is not installed; attempting download，使用最新的python版本之一，如pypy2.7-6.0要么pypy3.5-6.0。

JavaScript和Node.js支持 #

  • 对于构建指定language: node_js，nvm在构建时自动更新到最新版本。对于其他构建，已选择图像构建时的稳定版本，即0.33.11。
  • 预安装了以下NodeJS版本：11.0.0和8.12.0。

去支持 #

  • 预装Go：1.11.1
  • 通过使用指定语言版本，可以在构建期间安装其他Go版本go:-键。

JVM（Clojure，Groovy，Java，Scala）支持 #

  • 预安装的JVM：openjdk8，openjdk10，和openjdk11在x86上，默认

是openjdk8;openjdk7和openjdk8在ppc64le上。

  • 如果可以通过指定，可以获取其他JDK，包括Oraclejdk。
  • 下表总结了预安装的JVM工具版本：
