本指南涵盖的内容 #

本指南介绍了可用的软件包，工具和设置

Travis macOS CI环境（通常称为“CI环境”）。

概观 #

Travis CI在每次构建之前快照的虚拟机中运行构建

并在它的末尾回滚。这提供了许多好处：

  • 主机操作系统不受测试套件的影响
  • 运行之间没有状态持续存在
  • 无密码sudo可用
  • 测试套件可以通过安装各种服务brew

然后自定义这些服务的配置

每个构建的开始。

测试套件可用的环境称为特拉维斯CI

。

使用macOS #

要使用我们的macOS构建基础结构，请将以下内容添加到您的.travis.yml：

os: osx

家酿 #

每次更新虚拟机时都会安装和更新Homebrew。

该Travis Homebrew插件是安装依赖项的最简单，最快速和最可靠的方法。

Homebrew插件正确处理最新，过时和缺失的包。手动Homebrew依赖脚本容易出错，我们建议不要使用它们。

Homebrew插件默认使用构建映像上的Homebrew数据库，但可以配置为运行brew update如果需要的话。

环境变量 #

  • CI=true
  • TRAVIS=true
  • USER=travis
  • HOME=/Users/travis

此外，Travis CI设置您可以在构建中使用的环境变量，

例如标记构建，或运行构建后部署。

  • TRAVIS_BRANCH：对于不是由pull请求触发的构建，这是

当前正在建立的分支的名称;而对于构建触发

请求（在许多情况下，这将是master）。

  • TRAVIS_BUILD_DIR：存储库所在目录的绝对路径

  • TRAVIS_BUILD_ID
  • TRAVIS_BUILD_NUMBER
  • TRAVIS_COMMIT：当前版本正在测试的提交。

  • TRAVIS_COMMIT_RANGE

  • TRAVIS_JOB_ID
  • TRAVIS_JOB_NUMBER：当前作业的编号（例如，“4.1”）。
  • TRAVIS_PULL_REQUEST：如果当前作业是拉动，则拉取请求编号

如果不是拉取请求，请求“假”。

  • TRAVIS_SECURE_ENV_VARS：是否存在安全环境变量

用过的。该值为“true”或“false”。

  • TRAVIS_REPO_SLUGowner_name/repo_name）的

目前正在建立的资源库（例如，“travis-ci / travis-build”）。

  • TRAVIS_OS_NAME：在多操作系统版本中，此值指示作业正在运行的平台。

价值观是linux和osx

  • TRAVIS_TAG：如果当前构建用于git标记，则此变量将设置为标记的名称。
