使用Trusty #

要使用Ubuntu Trusty，请将以下内容添加到您的

.travis.yml。

dist: trusty

{：data-file =“。travis.yml”}

如果您想更多地了解使用的优缺点和当前状态

值得信赖，继续阅读。

Travis CI以隔离的方式运行每个构建Google Compute Engine为每个构建提供vanilla构建环境的虚拟机。

构建可以访问各种用于数据存储和消息传递的服务

可以安装他们运行所需的任何东西。

Linux基础架构 #

Travis CI在一个独立的Google Compute Engine虚拟机中运行每个构建，为每个构建提供一个vanilla构建环境。

这样做的好处是，构建之间不会出现任何状态，提供干净的平台并确保您的测试在从头开始构建的环境中运行。

您的构建会自动路由到此基础架构，您无需对其进行任何修改.travis.yml。

基于容器的基础设施 #

基于容器的基础设施已经完全弃用。

请删除任何sudo: false你的钥匙.travis.yml文件使用完全虚拟化Linux基础架构。

集团成员 #

执行构建的用户（$USER）属于一个派生自的主要群体

那个用户。

如果您需要修改组成员身份，请按以下步骤操作：

  1. 设置环境。这可以在构建生命周期中的任何时间完成

在构建脚本执行之前。

  • 设置和导出环境变量。
  • 加$USER到期望的二级组：sudo usermod -a -G SECONDARY_GROUP_1,SECONDARY_GROUP_2 $USER

您可以使用修改用户的主要组-g。

  1. 您的script看起来像是这样的：
script: sudo -E su $USER -c 'COMMAND1; COMMAND2; COMMAND3'

这会将环境变量传递给abash运行的过程

$USER，同时保留定义和属于的环境变量

上面给出的二级小组usermod。
