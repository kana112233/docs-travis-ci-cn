什么是持续集成（CI）？ #

持续集成是在小代码更改中合并的实践

经常 - 而不是在开发结束时合并一个大的变化

周期。目标是通过开发和测试更小的软件来构建更健康的软件

增量。这就是Travis CI的用武之地。

作为一个持续集成平台，Travis CI支持您的开发

通过自动构建和测试代码更改，立即提供流程

关于变革成功的反馈。 Travis CI还可以自动化其他部件

管理部署和通知的开发过程。

CI构建和自动化：构建，测试和部署 #

运行构建时，Travis CI会将您的GitHub存储库克隆为全新的存储库

虚拟环境，并执行一系列任务来构建和测试您的

码。如果其中一个或多个任务失败，则考虑构建

破碎。如果没有任何任务失败，那么构建就是

考虑通过，Travis CI可以部署您的代码

到Web服务器或应用程序主机。

CI构建还可以自动化交付工作流程的其他部分。这意味着

你可以让工作依赖于彼此建立阶段，

建立通知， 准备

部署在构建之后，还有许多其他任务。

建设，工作，阶段和阶段 #

在Travis CI文档中，一些常用词具有特定含义：

  • 相 - 顺序步骤

一份工作例如，install阶段，来到之前script相，

在可选之前deploy相。

  • 工作 - 将存储库克隆为虚拟的自动化流程

环境然后进行一系列的相比如编译你的

代码，运行测试等。如果返回代码，则作业失败script相

不为零。

  • 建立- 一群工作。例如，构建可能有两个工作，每一个

其中使用不同版本的编程语言测试项目。

一个建立完成所有工作后完成。

  • 阶段- 一群工作作为顺序构建的一部分并行运行

由多个组成的过程阶段。

打破构建 #

构建被考虑破碎当一个或多个工作完成时

状态不是通过：

  • 出错 - 一个命令before_install，install， 要么before_script

phase返回非零退出代码。工作立即停止。

  • 失败 - 一个命令scriptphase返回非零退出代码。该

作业继续运行直到完成。

  • 取消 - 用户在完成之前取消作业。

开始排除您的构建被破坏的原因。

##

Travis CI提供了一些不同的基础架构环境，因此您可以选择

最适合您项目的设置：

  • Ubuntu Linux - 这些Linux Ubuntu环境在完整虚拟机内运行，提供大量计算资源，并支持使用sudo，setuid，和setgid。
  • 苹果系统 - 使用macOS操作系统的几个版本之一。此环境对于构建需要macOS软件的项目非常有用，例如用Swift编写的项目。如果在macOS计算机上进行开发，则不要求使用macOS环境。
