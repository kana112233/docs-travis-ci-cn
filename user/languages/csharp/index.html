概观 #

C＃，F＃和Visual Basic项目的设置如下所示：

language: csharp
solution: solution-name.sln

{：data-file =“。travis.yml”}

什么时候可选solution密钥存在，Travis将运行NuGet包恢复并构建给定的解决方案。

选择运行时和版本来测试 #

单 #

默认情况下，Travis CI将使用最新的Mono版本。也可以针对特定版本的Mono测试项目。为此，请使用以下命令指定版本mono键入.travis.yml。例如，要测试最新的3.12.0和3.10.0：

language: csharp
mono:
  - latest
  - 3.12.0
  - 3.10.0
...

{：data-file =“。travis.yml”}

您可以选择以下Mono版本：

版本 已安装的软件包（仅限Linux，macOS始终包含所有内容）
3.10.0及更高版本 mono-complete，mono-vbnc，fsharp，nuget，referenceassemblies-pcl
3.8.0 mono-complete，mono-vbnc，fsharp，nuget
3.2.8 mono-complete，mono-vbnc，fsharp
2.10.8 mono-complete，mono-vbnc
禁用Mono（如果您只想要.NET Core，请使用此功能，请参阅下文）

注意：即使你指定例如： 3.12.0您的构建使用的版本实际上可能是3.12.1，具体取决于3.12.x系列中的最新版本（现在是Xamarin存储库的限制）。

Alpha，Beta和每周频道：安装和测试即将推出的Mono版本指定alpha，beta， 要么weekly作为版本号。请将您在这些频道上遇到的错误报告给Mono项目，以便在发布之前修复它们。

.NET核心 #

默认情况下，Travis CI不会针对.NET Core进行测试。要针对.NET Core进行测试，请将以下内容添加到您的.travis.yml。请注意，至少有一个script<command>是必需的，以建立。运用dotnet restore是一个很好的默认值。

language: csharp
mono: none
dotnet: 2.1.502
script:
 - dotnet restore
...

注意：您需要指定.NET Core SDK的版本号（不.NET Core Runtime）。

SDK的版本号可以在。上找到.NET核心网站。

针对Mono和.NET Core进行测试 #

您可以使用来测试Mono和.NET Corematrix.include。此示例针对最新的mono和.NET Core进行测试：

language: csharp
solution: travis-mono-test.sln

matrix:
  include:
    - dotnet: 2.1.502
      mono: none
      env: DOTNETCORE=2  # optional, can be used to take different code paths in your script
    - mono: latest
...

{：data-file =“。travis.yml”}

插件 #

该Coverity扫描不支持addon，因为它现在只适用于Windows上的msbuild。

运行单元测试（NUnit，xUnit等） #

要运行单元测试套件，首先需要安装测试运行器。推荐的方法是从NuGet安装它，因为这也适用于sudo

以下示例显示了您如何覆盖install和script安装测试运行器并将测试程序集传递给它以运行测试。

NUnit的 #

language: csharp
solution: solution-name.sln
install:
  - nuget restore solution-name.sln
  - nuget install NUnit.Console -Version 3.9.0 -OutputDirectory testrunner
script:
  - msbuild /p:Configuration=Release solution-name.sln
  - mono ./testrunner/NUnit.ConsoleRunner.3.9.0/tools/nunit3-console.exe ./MyProject.Tests/bin/Release/MyProject.Tests.dll

{：data-file =“。travis.yml”}

的xUnit #

language: csharp
solution: solution-name.sln
install:
  - nuget restore solution-name.sln
  - nuget install xunit.runners -Version 1.9.2 -OutputDirectory testrunner
script:
  - msbuild /p:Configuration=Release solution-name.sln
  - mono ./testrunner/xunit.runners.1.9.2/tools/xunit.console.clr4.exe ./MyProject.Tests/bin/Release/MyProject.Tests.dll

{：data-file =“。travis.yml”}

注意：有一个bug在Mono中，xUnit 2.0在测试执行后挂起，我们建议你坚持使用1.9.2直到它被修复。

使用解决方案级NuGet包 #

对于许多.NET项目，这将是找到的文件./.nuget/packages.config

nuget restore solution-name.sln然后也将安装该包。

language: csharp
solution: solution-name.sln
script:
  - msbuild /p:Configuration=Release solution-name.sln
  - mono ./packages/xunit.runners.*/tools/xunit.console.clr4.exe ./MyProject.Tests/bin/Release/MyProject.Tests.dll

{：data-file =“。travis.yml”}

注意使用文件名扩展（*）以避免必须硬编码测试运行器的版本。

该在针对.NET Core进行测试时支持。例：

language: csharp

dotnet: 2.1.502
solution: solution-name.sln

  - dotnet test
