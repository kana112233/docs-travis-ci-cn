指定要使用的Go版本 #

您可以使用任何带标记的Go版本x代替未成年人

或补丁级别使用最新的给定主要或次要版本，或使用

由…处理给我。

- 1.x
- "1.10"
- 1.11.x
- master

安装私有依赖项 #

go get使用HTTPS从GitHub而不是SSH来克隆依赖关系

我们需要一个不同的解决方法[

依赖](/user/private-dependencies)。

当通过HTTPS克隆时，git在封面下使用curl，而curl则允许

你要指定一个的.netrc

文件，您可以在其中存储特定的自定义身份验证凭据

域名，例如github.com。

去你的GitHub帐户和

创建个人访问令牌。

一定要给它repo范围，允许访问私人

库。

要减少令牌的访问权限，您还可以创建单独的用户

帐户只能访问特定项目所需的存储库。

复制令牌并将其存储在存储库中的.netrc中，具体如下

数据：

machine github.com
  login <username>
  password <token>

将其添加到您的存储库并将以下步骤添加到.travis.yml：

before_install:
- cp .netrc ~/.netrc
- chmod 600 ~/.netrc

{：data-file =“。travis.yml”}

默认构建脚本 #

Go项目假定默认使用Make或Go构建工具。在

如果存储库根目录中有Makefile，则默认命令为Travis CI

将用于运行您的项目测试套件

make

如果没有Makefile，它将是

go test ${gobuild_args} ./...

代替。

[一般建设

组态](/user/customizing-the-build/)指南。例如，添加

-v旗帜，覆盖script:.travis.yml

script: go test -v ./...

{：data-file =“。travis.yml”}

参数传递给默认值go test

gobuild_args:在配置的顶层，例如：

gobuild_args: -x -ldflags "-X main.VersionString v1.2.3"

{：data-file =“。travis.yml”}

go test -x -ldflags "-X main.VersionString v1.2.3" ./...

要通过运行没有参数的Scons来构建，请使用：

script: scons
