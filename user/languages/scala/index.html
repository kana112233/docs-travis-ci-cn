概观 #

Travis CI环境为JVM语言提供了大量的构建工具

多个JDK，Ant，Gradle，Maven和

指定Scala版本 #

要在构建中指定Scala版本：

language: scala
scala:
   - 2.9.3
   - 2.10.6
   - 2.11.11
   - 2.12.2

{：data-file =“。travis.yml”}

在Ubuntu Precise上，要使用Scala 2.12.X，您需要通过添加来启用Oracle JDK 8jdk: oraclejdk8到你的.travis.yml

使用sbt的项目 #

如果您的项目有projectbuild.sbt存储库中的文件

根，Travis CI使用sbt建立它。

谢谢paulp / SBT-演员sbt

动态检测并使用您的项目版本。

sbt依赖管理 #

Travis CI自动撤下sbt运行前的依赖关系

期间的测试script你的构建阶段。

sbt默认脚本命令 #

默认script命令是：

sbt ++$TRAVIS_SCALA_VERSION test

运行你的测试套件。

要使用不同的script命令，自定义

建立步骤。

自定义sbt参数 #

你可以覆盖sbt和JVM选项

通过传递额外的参数sbt。

例如，要运行compile和test使用不同的JVM参数：

script:
  - sbt -jvm-opts travis/jvmopts.compile ... compile
  - sbt -jvm-opts travis/jvmopts.test ... test

{：data-file =“。travis.yml”}

你也可以指定[额外

参数](https://github.com/paulp/sbt-extras#sbt–h)被传递给

使用默认构建脚本sbt_args关键在你的.travis.yml。

例如

sbt_args: -no-colors -J-Xss2m

{：data-file =“。travis.yml”}

会产生

script: sbt -no-colors -J-Xss2m ++$TRAVIS_SCALA_VERSION test

使用Gradle，Maven或Ant的项目 #

如果没有为sbt配置项目，则构建过程的行为类似于

典型Java项目。

使用Java 10和Up #

要使用OpenJDK和OracleJDK 10及更高版本进行测试，请参阅

Java文档。
