选择OTP版本进行测试 #

Travis CI VM提供64位Erlang OTPkerl。要指定要对项目进行测试的OTP版本，请使用otp_release键：

language: erlang
otp_release:
  - 18.2.1
  - 18.1
  - 18.0
  - 17.5
  - R16B03

{：data-file =“。travis.yml”}

通过添加获取VM上可用的预编译版本的完整列表kerl list installationsbefore_script:你的部分.travis.yml。请注意，此列表有不包括按需下载的版本，例如18.1。

默认测试脚本 #

默认情况下，Travis CI假定您的项目是使用构建的Rebar3并使用EUnit。 Erlang构建器默认使用的确切命令是

rebar3 eunit

如果你的项目有rebar.config要么Rebar.config存储库根目录中的文件。

在较旧的图像上rebar3不可用，我们回过头来rebar

rebar compile && rebar skip_deps=true eunit

如果没有rebar.config也不Rebar.config在存储库根目录中找到，Erlang构建器将回归到

make test

依赖管理 #

Travis CI上的Erlang构建器假设Rebar3用于依赖关系管理。

看到Rebar3文档了解更多详情。

在较旧的图像上rebar3不可用，我们回过头来rebar，并运行

rebar3 get-deps

安装rebar.config
