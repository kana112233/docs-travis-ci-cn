Elixir项目的CI环境 #

要选择Elixir VM，请在您的.travis.yml：

language: elixir

{：data-file =“。travis.yml”}

###

您可以指定Elixir版本来构建elixir键。

例如，

elixir: '1.5.2'

{：data-file =“。travis.yml”}

要么

elixir: '1.5'

{：data-file =“。travis.yml”}

后者指向最新的开发分支构建

有最新的补丁，但可能偶尔会被打破。

看到这个GitHub问题评论

更多细节。

指定OTP版本 #

请注意，Elixir对底层证券有要求

如果指定的OTP发布版本（隐式或显式）

如果不符合此要求，Travis CI将选择一个

为了你。

您还可以通过添加来覆盖此OTP版本选项otp_release。

例如：

language: elixir
elixir:
  - '1.2.2'
otp_release:
  - '18.2.1'

{：data-file =“。travis.yml”}

要测试具有不同OTP版本的多个Elixir版本：

language: elixir

elixir:
  - '1.0.5'
otp_release: '17.4'

matrix:
  include:
    - elixir: '1.2'
      otp_release: '18.0'

{：data-file =“。travis.yml”}

环境变量 #

正在使用的Elixir版本可用：

TRAVIS_ELIXIR_VERSION

与Erlang VM一样，正在使用的作业的OTP版本可用：

TRAVIS_OTP_RELEASE
