选择要测试的PHP版本 #

PHPUnit的。 Travis CI使用phpenv管理

虚拟机上安装了不同的PHP版本。

一个例子.travis.yml测试各种PHP版本的文件：

language: php
php:
  - '5.4'
  - '5.6'
  - '7.0'
  - hhvm # on Trusty only
  - nightly

{：data-file =“。travis.yml”}

X.Y版本是最近的别名X.Y.Z预安装在构建映像上的版本。

有关构建中使用的确切版本，请参阅构建日志中的“构建系统信息”。

这些可能不是最新版本。如果您需要确保使用最新版本，请提供第三个号码;例如。，

language: php
php:
  - 7.1.9

{：data-file =“。travis.yml”}

PHP 5.2（.x）和5.3（.x）支持仅适用于Precise #

我们不在Trusty上支持这些版本。

如果您需要测试它们，请使用Precise。

HHVM版本仅在Trusty上提供 #

Travis CI可以在Ubuntu Trusty上使用HHVM测试您的PHP应用程序：

php:
  - hhvm-3.18
  - hhvm-nightly

{：data-file =“。travis.yml”}

请注意，如果要在HHVM上运行PHPUnit，则必须在您的版本中明确安装5.7版.travis.yml由于HHVM和PHP7之间的兼容性问题：

before_script:
  - curl -sSfL -o ~/.phpenv/versions/hhvm/bin/phpunit https://phar.phpunit.de/phpunit-5.7.phar

{：data-file =“。travis.yml”}

自定义PHP配置 #

自定义PHP配置的最简单方法是使用phpenv config-add

before_script: phpenv config-add myconfig.ini

.-

和myconfig.ini

extension = "mongo.so"
date.timezone = "Europe/Paris"
default_socket_timeout = 120
# some other configuration directives...

.travis.yml：

before_script: echo 'date.timezone = "Europe/Paris"' >> ~/.phpenv/versions/$(phpenv version-name)/etc/conf.d/travis.ini

{：data-file =“。travis.yml”}

启用预安装的PHP扩展 #

您需要通过添加一个来启用它们extension="<extension>.so"换行到PHP配置文件（对于当前的PHP版本）。

最简单的方法是使用phpenv添加自定义配置文件，该文件启用并最终配置扩展：

before_install: phpenv config-add myconfig.ini

确保配置文件不以点开头（.）或连字符（-）因为这会阻止PHP加载您的自定义设置。

和myconfig.ini：

extension="mongo.so"
# some other mongo specific configuration directives
# or general custom PHP settings...

您也可以使用以下一行命令：

before_install: echo "extension = <extension>.so" >> ~/.phpenv/versions/$(phpenv version-name)/etc/php.ini

{：data-file =“。travis.yml”}

禁用预安装的PHP扩展 #

before_script:
  - phpenv config-rm xdebug.ini

{：data-file =“。travis.yml”}

安装其他PHP扩展 #

可以在Travis CI环境中安装自定义PHP扩展

运用PECL，但它们必须针对PHP构建

正在测试的版本。

例如，要安装memcache：

pecl install <extension>

PECL将在安装结束时自动启用扩展。如果

你想配置你的扩展，使用phpenv config-add命令添加

您的before_script中的自定义ini配置文件。

也可以“手动”进行安装，但您必须这样做

安装后手动启用扩展程序phpenv config-add和自定义ini文件或使用此一行命令：

echo "extension=<extension>.so" >> ~/.phpenv/versions/$(phpenv version-name)/etc/php.ini

另见。

如果您需要特定版本的预安装扩展，则需要强制使用

安装特定版本-f旗。例如：

pecl install -f mongo-1.2.12

Apache + PHP #

目前Travis CI不支持mod_php对于apache，但你可以配置

php-fpm对于您的集成测试：

before_script:
  - sudo apt-get update
  - sudo apt-get install apache2 libapache2-mod-fastcgi
  # enable php-fpm
  - sudo cp ~/.phpenv/versions/$(phpenv version-name)/etc/php-fpm.conf.default ~/.phpenv/versions/$(phpenv version-name)/etc/php-fpm.conf
  - sudo a2enmod rewrite actions fastcgi alias
  - echo "cgi.fix_pathinfo = 1" >> ~/.phpenv/versions/$(phpenv version-name)/etc/php.ini
  - sudo sed -i -e "s,www-data,travis,g" /etc/apache2/envvars
  - sudo chown -R travis:travis /var/lib/apache2/fastcgi
  - ~/.phpenv/versions/$(phpenv version-name)/sbin/php-fpm
  # configure apache virtual hosts
  - sudo cp -f build/travis-ci-apache /etc/apache2/sites-available/000-default.conf
  - sudo sed -e "s?%TRAVIS_BUILD_DIR%?$(pwd)?g" --in-place /etc/apache2/sites-available/000-default.conf
  - sudo service apache2 restart

{：data-file =“。travis.yml”}

注意sudo不适用于正在运行的构建基于容器的。

你需要拥有build/travis-ci-apache将配置您的文件

虚拟主机像往常一样，php-fpm的重要部分是这样的：

<VirtualHost *:80>
  # [...]

  DocumentRoot %TRAVIS_BUILD_DIR%

  <Directory "%TRAVIS_BUILD_DIR%/">
    Options FollowSymLinks MultiViews ExecCGI
    AllowOverride All
    Require all granted
  </Directory>

  # Wire up Apache to use Travis CI's php-fpm.
  <IfModule mod_fastcgi.c>
    AddHandler php5-fcgi .php
    Action php5-fcgi /php5-fcgi
    Alias /php5-fcgi /usr/lib/cgi-bin/php5-fcgi
    FastCgiExternalServer /usr/lib/cgi-bin/php5-fcgi -host 127.0.0.1:9000 -pass-header Authorization

    <Directory /usr/lib/cgi-bin>
        Require all granted
    </Directory>
  </IfModule>

  # [...]
</VirtualHost>
