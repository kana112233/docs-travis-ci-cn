选择Haxe版本进行测试 #

Travis CI的Haxe工作人员下载并安装了Haxe的二进制文件。

要选择一个或多个版本，请使用haxe:

关键在你的.travis.yml文件，例如：

language: haxe
haxe:
  - "3.2.1"
  - stable      # the latest stable release defined in https://haxe.org/download/list/
  - development # the latest build of the development branch in http://build.haxe.org/

{：data-file =“。travis.yml”}

默认Haxe版本 #

如果你离开了haxe:钥匙出你的.travis.yml，Travis CI将使用stable，是的

中定义的最新稳定版本haxe.org下载列表。

默认Neko版本 #

默认情况下，ネ2.2.0也将下载并安装。

使用neko:关键在你的.travis.yml文件以指定不同的Neko版本，

例如：

language: haxe
neko: "2.0.0"

{：data-file =“。travis.yml”}

然而，不像haxe:，你只能提供一个值（不是数组）neko:。

测试配置 #

如果您的项目使用标准hxml文件进行构建，则可以指定

使用的hxml文件列表hxml:键，例如：

language: haxe

  - build.hxml

{：data-file =“。travis.yml”}
