本指南涵盖的内容 #

本指南介绍了Android项目特有的构建环境和配置主题。请务必阅读我们的教程和一般构建配置首先指导。

Android构建在macOS环境中不可用。

Android项目的CI环境 #

概观 #

Travis CI环境为JVM语言提供了大量的构建工具多个JDK，Ant，Gradle，Maven，SBT和Leiningen。

通过设置

language: android
dist: trusty

{：data-file =“。travis.yml”}

在你的.travis.yml

Android版本仅在我们的Trusty图像上受支持，因此您需要明确指定dist: trusty在.travis.yml文件中。

.travis.yml

language: android
dist: trusty
android:
  components:
    # Uncomment the lines below if you want to
    # use the latest revision of Android SDK Tools
    # - tools
    # - platform-tools

    # The BuildTools version used by your project
    - build-tools-26.0.2

    # The SDK version used to compile your project
    - android-26

    # Additional components
    - extra-google-google_play_services
    - extra-google-m2repository
    - extra-android-m2repository

    # Specify at least one system image,
    # if you need to run emulator(s) during your tests
    - sys-img-x86-android-26
    - sys-img-armeabi-v7a-android-17

{：data-file =“。travis.yml”}

###

在你的.travis.yml

language: android
dist: trusty
android:
  components:
    - build-tools-26.0.2
    - android-26
    - extra

{：data-file =“。travis.yml”}

必须指定确切的组件名称（过滤别名，如add-on要么extrasdkmanager --list（最好是在你当地的开发机器上）。

处理许可证 #

默认情况下，Travis CI将接受所有请求的许可证，但也可以定义要接受的白色许可证列表，如以下示例所示：

language: android
dist: trusty
android:
  components:
    - build-tools-26.0.2
    - android-26
    - add-on
    - extra
  licenses:
    - 'android-sdk-preview-license-52d11cd2'
    - 'android-sdk-license-.+'
    - 'google-gdk-license-.+'

{：data-file =“。travis.yml”}

为了获得更大的灵活性，还可以使用正则表达式引用许可证（使用Tcl语法作为expect命令用于自动响应交互式提示）。

预装组件 #

虽然预装了以下组件，但确切的列表可能会更改，恕不另行通知。为确保构建环境的稳定性，建议您明确指定项目所需的组件。

  - android-wait-for-emulator

依赖管理 #

Travis CI Android构建器假定您的项目是使用Maven或Gradle等JVM构建工具构建的，它将在运行测试之前自动降低项目依赖性，而无需您付出任何努力。

如果您的项目是使用Ant或任何其他不自动处理依赖项的构建工具构建的，则需要指定要使用的确切命令install:关键在你的.travis.yml， 例如：

language: android
dist: trusty
install: ant deps

{：data-file =“。travis.yml”}

Maven的默认测试命令 #

如果你的项目有pom.xml存储库根目录中的文件但没有build.gradle，Maven 3将用于构建它。默认情况下它会使用

mvn install -B

运行你的测试套件。这可以被覆盖，如下所述一般构建配置指南。

Gradle的默认测试命令 #

如果你的项目有build.gradle在存储库根目录中，Gradle将用于构建它。默认情况下它会使用

gradle build connectedCheck

运行你的测试套件。如果您的项目还包括gradlew在存储库根目录中的包装器脚本，Travis Android构建器将尝试使用它。默认命令将变为：

./gradlew build connectedCheck

这可以被覆盖，如下所述一般构建配置指南。

高速缓存 #

Gradle中依赖性缓存的特性意味着要避免在每次构建后上传缓存，您需要将以下行添加到您的.travis.yml：

before_cache:
  - rm -f  $HOME/.gradle/caches/modules-2/modules-2.lock
  - rm -fr $HOME/.gradle/caches/*/plugin-resolution/
cache:
  directories:
    - $HOME/.gradle/caches/
    - $HOME/.gradle/wrapper/
    - $HOME/.android/build-cache

{：data-file =“。travis.yml”}

默认测试命令 #

如果Travis CI无法检测到Maven或Gradle文件，Travis CI Android构建器将尝试使用Ant来构建您的项目。默认情况下它会使用

ant debug install test

运行你的测试套件。这可以被覆盖，如下所述一般构建配置指南。
