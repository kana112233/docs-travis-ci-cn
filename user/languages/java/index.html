概观 #

Travis CI环境包含各种版本的OpenJDK，Oracle JDK，

Gradle，Maven和Ant。

要使用Java环境，请将以下内容添加到您的.travis.yml：

language: java

{：data-file =“。travis.yml”}

使用Maven的项目 #

Maven依赖管理 #

在运行构建之前，Travis CI会安装依赖项：

mvn install -DskipTests=true -Dmaven.javadoc.skip=true -B -V

或者如果你的项目使用了mvnw包装脚本：

./mvnw install -DskipTests=true -Dmaven.javadoc.skip=true -B -V

请注意，Travis CI构建生命周期和Maven构建生命周期使用类似

不同构建阶段的术语。例如，install在特拉维斯CI

构建比…更早install在Maven构建生命周期中。更多细节

可以找到关于Travis构建生命周期

和Maven构建生命周期。

Maven默认脚本命令 #

如果你的项目有pom.xml存储库根目录中的文件但没有build.gradle，

Travis CI使用Maven 3构建您的项目：

mvn test -B

如果您的项目还包括mvnw存储库根目录中的包装脚本，

Travis CI使用它代替：

./mvnw test -B

默认命令不生成JavaDoc（-Dmaven.javadoc.skip=true）。

要使用不同的script命令，自定义建立步骤。

使用Gradle的项目 #

Gradle依赖管理 #

在运行构建之前，Travis CI会安装依赖项：

gradle assemble

要么

./gradlew assemble

要使用不同的install命令，自定义安装步骤。

Gradle默认脚本命令 #

如果您的项目包含build.gradleTravis CI中的存储库根目录中的文件

使用Gradle构建项目：

gradle check

如果您的项目还包括gradlew存储库中的包装脚本

root，Travis CI使用该包装：

./gradlew check

要使用不同的script命令，自定义建立步骤。

高速缓存 #

Gradle中依赖性缓存的特性意味着要避免上传

在每次构建之后缓存，您需要将以下行添加到您的

.travis.yml：

before_cache:
  - rm -f  $HOME/.gradle/caches/modules-2/modules-2.lock
  - rm -fr $HOME/.gradle/caches/*/plugin-resolution/
cache:
  directories:
    - $HOME/.gradle/caches/
    - $HOME/.gradle/wrapper/

{：data-file =“。travis.yml”}

请注意，如果您使用Gradlesudo（即sudo ./gradlew assemble），上面的缓存配置将没有任何效果，因为依赖项将在/root/.gradle哪个travis用户帐户没有写入权限。

使用Ant的项目 #

蚂蚁依赖管理 #

因为没有单一的标准方法来安装项目依赖项

Ant，您需要指定要使用的确切命令install:关键在你的

.travis.yml， 例如：

language: java
install: ant deps

{：data-file =“。travis.yml”}

Ant默认脚本命令 #

如果Travis CI没有检测到Maven或Gradle文件，它会运行Ant：

ant test

要使用不同的script命令，自定义建立步骤。

针对多个JDK进行测试 #

要测试多个JDK，请使用jdk:键入.travis.yml。例如，

测试Oracle JDK 8和9，以及OpenJDK 8：

jdk:
  - oraclejdk8
  - oraclejdk9
  - openjdk8

{：data-file =“。travis.yml”}

请注意，macOS不支持针对多个Java版本进行测试。看到

该macOS构建环境更多

细节。

不同dists的可用JVM列表位于

  • 已安装JDKXenial
  • 已安装JDK可靠
  • 已安装JDK精确

在一个作业中切换JDK（Java 8及更低版本） #

如果您的构建需要在作业期间切换JDK（Java 8及更低版本），则可以使用

jdk_switcher use …。

script:
  - jdk_switcher use oraclejdk8
  - # do stuff with Java 8
  - jdk_switcher use openjdk8
  - # do stuff with open Java 8

{：data-file =“。travis.yml”}

用于jdk_switcher也更新$JAVA_HOME适当。

将Oracle JDK 8更新到最新版本 #

您的存储库可能需要比预安装版本更新的Oracle JDK版本

版。

（你可以咨询已发布的Oracle JDK包列表。）

以下示例将使用最新的Oracle JDK 8：

addons:
  apt:
    packages:
      - oracle-java8-installer

{：data-file =“。travis.yml”}

使用Java 10及更高版本 #

请注意oraclejdk10自2018年10月起是EOL，因此Travis CI不再支持EOL。

看到https://www.oracle.com/technetwork/java/javase/eol-135779.html{：data-proofer-ignore =“”}。

Linux上支持OracleJDK 11及更高版本

使用Linux和macOS支持OpenJDK 10及更高版本

install-jdk.sh。

jdk:
  - oraclejdk8
  - oraclejdk11
  - openjdk10
  - openjdk11

{：data-file =“。travis.yml”}

在一个作业中切换JDK（到Java 10及更高版本） #

如果您的构建需要在作业期间切换JDK（Java 8及更高版本），则可以使用

install-jdk.sh。

jdk: openjdk10
script:
  - jdk_switcher use openjdk10
  - # do stuff with OpenJDK 10
  - export JAVA_HOME=$HOME/openjdk11
  - $TRAVIS_BUILD_DIR/install-jdk.sh --install openjdk11 --target $JAVA_HOME
  - # do stuff with open OpenJDK 11

{：data-file =“。travis.yml”}
