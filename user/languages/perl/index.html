指定Perl版本 #

Travis CI使用Perlbrew提供几个Perl版本

你可以测试你的项目：

language: perl
perl:
  - "5.30"
  - "5.28"
  - "5.26"

{：data-file =“。travis.yml”}

这些版本由major.minor数字是精确补丁的别名

水平，可能会有变化。对于预安装的精确版本

VM，请参阅构建日志中的“构建系统信息”。

不支持早于5.8的Perl版本。

具有线程支持的Perl运行时 #

Some Perls have been compiled with threading support. They have been compiled with the additional compile flags -Duseshrplib and -Duseithreads:

5.26-shrplib
5.24-shrplib
5.22-shrplib
5.20-shrplib
5.18-shrplib

默认构建脚本 #

默认构建脚本根据您的项目而有所不同：

  • 如果您的存储库有Build.PL在根中： ``` perl Build.PL && ./Build test

* 如果你的存储库在根目录中有Makefile.PL：

perl Makefile.PL && make test


* 如果两者都找不到：

make test



## 依赖管理

默认情况下Travis CI使用`cpanm`管理项目的依赖项。

cpanm –quiet –installdeps –notest .

```

覆盖构建命令时，请勿使用sudo #

什么时候覆盖install:调整依赖安装命令的关键（for

例如，要运行带有详细标志的cpanm），请不要使用sudo。特拉维斯CI

环境通过Perlbrew在非特权用户中安装Perls$HOME

目录。运用sudo将导致意外安装依赖项

（对于Travis CI Perl构建器）位置，它们不会加载。

构建矩阵 #

对于Perl项目，env和perl可以作为数组给出

构建构建矩阵。

环境变量 #

作业正在使用的Perl版本可用TRAVIS_PERL_VERSION。
