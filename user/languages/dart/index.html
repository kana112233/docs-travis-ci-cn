选择Dart版本进行测试 #

Travis CI上的Dart工作人员下载并安装Dart SDK档案。看到

该Dart下载档案列表

可用的档案。默认情况下，会下载最新的稳定SDK版本。至

明确选择一个或多个版本，使用dart键。例如：

language: dart
dart:
# Install the latest stable release
- stable
# Install the latest dev release
- dev
# Install a specific stable release - 1.15.0
- "1.15.0"
# Install a specific dev release, using a partial download URL - 1.16.0-dev.3.0
- "dev/release/1.16.0-dev.3.0"

{：data-file =“。travis.yml”}

运行测试 #

如果您的包裹取决于test包，pub run test将会

默认运行。这通常只在Dart VM上运行测试，但您可以

配置它默认情况下在其他平台上运行。

您还可以使用自定义Travis传递给测试运行器的参数

dart_task在.travis.yml。

language: dart
dart_task:
- test: --platform vm
- test: --platform chrome

{：data-file =“。travis.yml”}

可用的浏览器 #

Travis在Linux和Safari上默认安装了Firefox和Chrome

在macOS上。但是，如果您想在Dartium上运行测试，则需要

通过添加安装它install_dartium: true无论是在顶层还是在

特别的任务。

language: dart
dart_task:
- test: --platform vm
- test: --platform dartium
  install_dartium: true

{：data-file =“。travis.yml”}

XVFB #

在Linux上，测试运行器使用默认情况下，它可以使用浏览器

像Chrome需要显示器。但是，这可能会干扰某些

应用程序，因此您可以通过设置将其关闭xvfb: false无论是在顶部

language: dart
dart_task:
- test: --exclude-tags no-xvfb
- test: --tags no-xvfb
  xvfb: false

{：data-file =“。travis.yml”}

XVFB从未在macOS上使用过，因为它不使用X windows系统。

其他任务 #

除了运行测试之外，还有几个任务可用。

###

运行验证您的代码没有任何静态

dartanalyzer: true。默认情况下，它会分析所有Dart

true。

language: dart
dart_task:

# As long as you don't want any other configuration, you can just use the name
# of a task instead of "name: true".
- test

# Warnings are fatal, but we only analyze the lib/ directory.
- dartanalyzer: --fatal-warnings lib

{：data-file =“。travis.yml”}

格式化 #

运行Dart格式化程序

dartfmt: true。如果您的包裹取决于

dart_style

它会使用dartfmt

language: dart
dart_task:
- test: --platform vm
- test: --platform chrome
- dartfmt

{：data-file =“。travis.yml”}

环境变量 #

  • 作业正在使用的Dart版本可用TRAVIS_DART_VERSION。
  • TRAVIS_DART_TEST将会true如果当前任务使用test。
  • TRAVIS_DART_ANALYZE将会true如果当前任务使用dartanalyzer。
  • TRAVIS_DART_FORMAT将会true如果当前任务使用dartfmt。
