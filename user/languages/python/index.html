Python 3.7及更高版本 #

你需要添加dist: xenial到你的.travis.yml文件使用Python 3.7及更高版本。

例如：

dist: xenial   # required for Python >= 3.7
language: python
python:
  - "3.7"
  - "3.7-dev"  # 3.7 development branch
  - "3.8-dev"  # 3.8 development branch
  - "nightly"  # nightly build

{：data-file =“。travis.yml”}

特拉维斯CI使用孤立的虚拟现实 #

CI环境为每个Python使用单独的virtualenv实例

版。这意味着只要您指定language: python在.travis.yml你的测试将在virtualenv中运行（无需你明确地创建它）。

系统Python不使用，不应该依赖。如果你需要

安装Python包，通过pip而不是apt来做。

如果您决定使用apt，请注意，出于兼容性原因，您只能使用Ubuntu中可用的默认Python版本（例如，对于Trusty，这意味着2.7.6和3.4.3）。

要访问virtualenv中的包，您需要指定它应该使用--system-site-packages选项。

为此，请在您的帐户中添加以下内容.travis.yml：

language: python
python:
  - "2.7"
  - "3.4"
virtualenv:
  system_site_packages: true

{：data-file =“。travis.yml”}

PyPy支持 #

Travis CI支持PyPy和PyPy3。

要针对PyPy测试项目，请将“pypy3.5”添加到Pythons列表中

在你的.travis.yml：

language: python
python:
  - "2.7"
  - "3.5"
  - "3.6"
  # PyPy versions
  - "pypy3.5"
# command to install dependencies
install:
  - pip install -r requirements.txt
  - pip install .
# command to run tests
script: pytest

{：data-file =“。travis.yml”}

使用Tox作为构建脚本 #

由于Travis的设计方式，与之互动TOX并不简单。

如上所述以上，Travis已经在隔离的virtualenv中运行测试language: python已指定，因此请在创建更多带有tox的环境时牢记这一点。如果您希望在Travis创建的virtualenv之外运行tox，那么使用它可能是一个更好的主意language: generic代替language: python。

如果你使用tox来测试你的代码对多个版本的python，你有两个选择：

  • 使用language: generic并在运行tox之前手动安装你感兴趣的python版本（没有手动安装，tox只能访问默认的Ubuntu python版本–Telmy的2.7.6和3.4.3）
  • 使用language: python和一个构建矩阵，为每个分支使用不同版本的python（您可以使用。指定python版本python键）。这将确保安装您感兴趣的版本并使您的工作负载并行化。

一个很好的例子travis.yml使用Travis构建矩阵运行tox是扭曲/克莱因。

在多个操作系统上运行Python测试 #

.travis.yml

language: python            # this works for Linux but is an error on macOS or Windows
matrix:
  include:
    - name: "Python 3.7.1 on Xenial Linux"
      python: 3.7           # this works for Linux but is ignored on macOS or Windows
      dist: xenial          # required for Python >= 3.7
    - name: "Python 3.7.2 on macOS"
      os: osx
      osx_image: xcode10.2  # Python 3.7.2 running on macOS 10.14.3
      language: shell       # 'language: python' is an error on Travis CI macOS
    - name: "Python 3.7.3 on Windows"
      os: windows           # Windows 10.0.17134 N/A Build 17134
      language: shell       # 'language: python' is an error on Travis CI Windows
      before_install: choco install python
      env: PATH=/c/Python37:/c/Python37/Scripts:$PATH
install: pip3 install --upgrade pip  # all three OSes agree about 'pip3'
# 'python' points to Python 2.7 on macOS but points to Python 3.7 on Linux and Windows
# 'python3' is a 'command not found' error on Windows but 'py' works on Windows only
script: python3 my_app.py || python my_app.py

如果你需要测试Django的多个版本，你可以指导

Travis CI使用不同的环境变量集或值进行多次运行。

使用ENV键入.travis.yml文件，例如：

env:
  - DJANGO_VERSION=1.10.8
  - DJANGO_VERSION=1.11.5

{：data-file =“。travis.yml”}

然后在依赖项安装脚本中使用ENV变量值进行测试

案例或测试脚本参数值。这里我们使用ENV变量值来指示

pip来安装一个确切的版本：

install:
  - pip install -q Django==$DJANGO_VERSION
  - python setup.py -q install

{：data-file =“。travis.yml”}

相同的技术通常用于针对多个数据库等测试项目。

有关真实世界的示例，请参阅getsentry /哨兵和jpvanhal /烧瓶分割。
