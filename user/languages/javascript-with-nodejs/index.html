依赖管理 #

Travis CI使用NPM要么安装项目依赖项。

请注意，Travis CI环境中默认没有安装npm软件包。

运用npm #

使用具体的npm#### 版 #

将以下内容添加到before_install相的.travis.yml：

before_install:
  - npm i -g npm@version-number

{：data-file =“。travis.yml”}

npm ci### 支持

如果package-lock.json要么npm-shrinkwrap.json存在和你的npm版本

支持它，Travis CI将使用npm cinpm install。

此命令将删除您的node_modules文件夹并安装所有依赖项

在您的锁文件中指定。

运用yarn #

特拉维斯CI检测使用纱。

如果两者package.json和yarn.lock存在于当前

目录，我们运行以下命令代替

npm install：

yarn

注意yarn需要Node.js版本4或更高版本。

如果工作不符合此要求，npm install用来

代替。

使用具体的yarn#### 版 #

将以下内容添加到before_install相的.travis.yml：

before_install:
  - curl -o- -L https://yarnpkg.com/install.sh | bash -s -- --version version-number
  - export PATH="$HOME/.yarn/bin:$PATH"

{：data-file =“。travis.yml”}

正如所示yarn cache dir。

如果您的缓存需要包含其他指令，您可以使用：

cache:
  yarn: true

有关更多信息，请参阅高速缓存文档。

Ember Apps #

您可以在Travis CI上构建您的Ember应用程序。默认测试框架

是Qunit。以下示例显示了如何构建和

测试不同的Ember版本。

dist: trusty
addons:
  apt:
    sources:
      - google-chrome
    packages:
      - google-chrome-stable
language: node_js
node_js:
  - "7"
env:
    - EMBER_VERSION=default
    - EMBER_VERSION=release
    - EMBER_VERSION=beta
    - EMBER_VERSION=canary
matrix:
  fast_finish: true
  allow_failures:
    - env: EMBER_VERSION=release
    - env: EMBER_VERSION=beta
    - env: EMBER_VERSION=canary

before_install:
    # setting the path for phantom.js 2.0.0
    - export PATH=/usr/local/phantomjs-2.0.0/bin:$PATH
    # starting a GUI to run tests, per https://docs.travis-ci.com/user/gui-and-headless-browsers/#using-xvfb-to-run-tests-that-require-a-gui
    - export DISPLAY=:99.0
    - sh -e /etc/init.d/xvfb start
    - "npm config set spin false"
    - "npm install -g npm@^2"
install:
    - mkdir travis-phantomjs
    - wget https://s3.amazonaws.com/travis-phantomjs/phantomjs-2.0.0-ubuntu-12.04.tar.bz2 -O $PWD/travis-phantomjs/phantomjs-2.0.0-ubuntu-12.04.tar.bz2
    - tar -xvf $PWD/travis-phantomjs/phantomjs-2.0.0-ubuntu-12.04.tar.bz2 -C $PWD/travis-phantomjs
    - export PATH=$PWD/travis-phantomjs:$PATH
    - npm install -g bower
    - npm install
    - bower install
script:
    - ember test --server

{：data-file =“。travis.yml”}

Node.js v4（或io.js v3）编译器要求 #

要为io.js v3或Node.js v4或更高版本编译本机模块，请执行以下操作：[C ++ 11

标准](https://en.wikipedia.org/wiki/C%2B%2B11)兼容编译器

需要。更具体地说，gcc 4.8（或更高版本）或clang 3.5（或更高版本）

作品。

我们的Trusty图像具有满足此要求的gcc和clang，但是精确

图像没有。

例如，要将这些编译器更新为更新的版本，gcc/g++到版本

4.8，在你的添加中添加以下内容.travis.yml：

language: node_js
node_js:
  - "4"
env:
  - CXX=g++-4.8
addons:
  apt:
    sources:
      - ubuntu-toolchain-r-test
    packages:
      - g++-4.8

{：data-file =“。travis.yml”}
