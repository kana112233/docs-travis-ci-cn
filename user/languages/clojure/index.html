依赖管理 #

如果您使用Leiningen，它将自动安装任何依赖项

列在project.clj文件。

备用安装步骤 #

如果您需要在运行测试之前安装其他依赖项，那么您应该这样做

设置正确的:hooks在project.clj。

如果由于某种原因你不能使用钩子，你可以覆盖你的安装步骤

.travis.yml：

install: lein protobuf install

{：data-file =“。travis.yml”}

见构建配置指南了解更多。

构建脚本 #

使用Midje #

如果您的项目使用Midje， 确保

雷音，midje是你的

project.clj开发依赖项列表和覆盖script:在

.travis.yml运行Midje任务：

script: lein midje

{：data-file =“。travis.yml”}

为Leiningen 1添加:dev-dependencies至project.clj：

:dev-dependencies [[midje "1.4.0"]
                   [lein-midje "1.0.10"]])

{：data-file =“。project.clj”}

Leiningen 2取代了:dev-dependencies与个人资料：

:profiles {:dev {:dependencies [[midje "1.6.3"]]
                 :plugins [[lein-midje "3.0.0"]]}}

{：data-file =“。project.clj”}

请注意，对于仅支持Clojure 1.3.0及更高版本的项目

版本，您可能需要排除瞬态org.clojure/clojure对于Midje in

project.clj：

:dev-dependencies [[midje "1.4.0" :exclusions [org.clojure/clojure]]
                   [lein-midje "1.0.10"]])

{：data-file =“。project.clj”}

对于现实世界的例子，请参阅Knockbox。

在特拉维斯CI上使用Speclj #

如果您的项目使用Speclj，确保它是

在您的开发依赖项中列出project.clj，包括这个

script:排队.travis.yml：

script: lein spec

{：data-file =“。travis.yml”}

对于Leiningen 1，Speclj应列在下面:dev-dependencies在project.clj：

:dev-dependencies [[speclj "3.3.1"]]

{：data-file =“。project.clj”}

Leiningen 2取代了:dev-dependencies与个人资料：

:profiles {:dev {:dependencies [[speclj "3.3.1"]]}}

{：data-file =“。project.clj”}

使用Leiningen 1 #

Leiningen 1与2.4.x并排提供。要使用它，请指定lein键入.travis.yml：

lein: lein1

{：data-file =“。travis.yml”}

如果你需要使用lein二进制文件before_script，install:，script:命令等，使用lein1：

before_install:
  - lein1 bootstrap

{：data-file =“。travis.yml”}

任务链需要使用do任务：

script: lein1 do javac, test

{：data-file =“。travis.yml”}

针对多种版本的Clojure进行测试 #

与Leiningen 1 #

Leiningen有一个很棒的插件叫做雷音的多这让你可以毫不费力地测试多个版本

Clojure（例如，最新开发版本的1.3,1.4和alphas / beta / snapshots）。因为leiningen可以对任何版本的Clojure进行测试（不一定与Leiningen本身使用的版本相同），

Clojure不需要运行时切换器（如RVM）。

要在Travis CI上使用lein-multi，请将其添加到:plugins在project.clj中（注意，此功能仅从Leiningen 1.7.0开始提供）和

覆盖script:运行lein multi test而不是默认lein test：

language: clojure
script: lein1 multi test

{：data-file =“。travis.yml”}

有关真实世界的示例，请参阅贩子。

与Leiningen 2 #

Leiningen 2具有替代的核心功能lein-multi：简介。运行您的测试

使用多个配置文件（因此，多个依赖集或Clojure版本）lein with-profile这样的命令：

lein: lein
script: lein with-profile dev:1.4 test

{：data-file =“。travis.yml”}

哪里dev:1.4是一个以冒号分隔的要运行的配置文件列表test任务反对。使用lein profiles列出项目的个人资料

和lein help with-profile了解更多关于with-profiles任务。

使用更新版本的Leiningen #

如果您的Clojure项目需要更新版本的Leiningen，您可以使用以下命令指定：

language: clojure

lein: 2.6.1 # version 2 and up

{：data-file =“。travis.yml”}

如果没有预先安装，该作业将安装指定版本的Leiningen，

然后继续安装项目的依赖项。
