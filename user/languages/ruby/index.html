指定Ruby版本和实现 #

Travis CI上的Ruby环境使用RVM提供很多

Ruby实现，版本甚至补丁级别。

要指定它们，请使用rvm:关键在你的.travis.yml文件：

language: ruby
rvm:
  - 2.2
  - jruby
  - truffleruby
  - 2.0.0-p247

{：data-file =“。travis.yml”}

请注意rvm:key仅在Ruby Build Environments中可用，而不是在

包含ruby实现的其他图像。

当我们升级RVM和Rubies时，别名就像2.2要么jruby指向

不同的确切版本和补丁级别。

运用.ruby-version #

如果没有指定ruby版本rvm关键，Travis CI使用了

版本中指定的版本.ruby-version文件在存储库的根目录中

一个是可用的。

Rubinius的 #

如果您使用的是macOS或Trusty环境，也可以使用

到你的.travis.yml，其中X.Y.Z指定列出的Rubinius版本

language: ruby

Bundler 2.0 #

2019年1月3日，Bundler团队发布了Bundler 2.0

它放弃了对2.2及更早版本的Ruby版本的支持，并添加了一个新的依赖项

在RubyGems 3.0.0上。

随后的发布，2.0.1，

需要RubyGems 2.5.0。

在许多配置下，Travis CI即时安装Ruby运行时。

这意味着在运行时安装最新的Bundler，这可能会导致问题

由于不满意的要求。

如果由于“未安装bundler”错误而发现构建失败，请尝试以下解决方案之一：

  • 如果您使用的是Ruby 2.3或更高版本，并且希望升级到Bundler 2.0，

在你的。中使用以下内容.travis.yml更新RubyGems：

before_install:
  - gem update --system
  - gem install bundler

{：data-file =“。travis.yml”}

  • 如果您使用的是Ruby 2.3.x但希望继续使用Bundler 1.x（例如，依赖于

Rails 4.2.x）等原因，写道：

before_install:
  - gem uninstall -v '>= 2' -i $(rvm gemdir)@global -ax bundler || true
  - gem install bundler -v '< 2'

{：data-file =“。travis.yml”} 该gem uninstall上面的命令删除安装在中的任何Bundler 2.x.

在Ruby运行时安装期间RVM的“全局”gemset，它将是

被选为默认值bundle命令。

我们忽略了该命令的失败，因为最有可能失败

意味着没有匹配的Bundler版本可以卸载。

您的构建配置可能需要这些变通办法的组合。

缓存Bundler #

Bundler安装可能需要一段时间，从而减慢构建速度。你可以说

Travis CI缓存已安装的捆绑包。

在第一次构建时，我们加热缓存。在第二个，我们将拉入

缓存，制作bundle install只需几秒钟就可以运行。

通过排除非必要依赖项来加速构建 #

很多项目都包括像ruby-debug，unicorn要么newrelic_rpm

在他们的默认宝石集中。

这会大大减慢安装过程，通常是那些

运行测试时不需要库。这还包括图书馆

编译本机代码，减慢安装和整体测试时间

更。

最重要的是，隐含地插入的库ruby_core_source要么

linecache19，当Travis CI升级Ruby版本时，一定会失败

patchlevels。

对于你在生产中只需要的宝石也是如此，比如Unicorn

New Relic库等。

您可以通过将这些库移动到a来加快安装过程

您的Gemfile中的单独部分，例如production：

group :production do
  gem 'unicorn'
  gem 'newrelic_rpm'
end

调整Bundler参数以明确排除此组：

bundler_args: --without production

{：data-file =“。travis.yml”}

享受更快的构建，这也不太容易出现编译问题。

自定义Bundler参数和Gemfile位置 #

默认的Gemfile位置是Gemfile在项目的根目录中。

要指定自定义Gemfile名称或位置：

gemfile: gemfiles/Gemfile.ci

{：data-file =“。travis.yml”}

如果以这种方式指定Gemfile的位置，则如果找不到该文件，则构建将失败。

你可以通过额外的论点

至bundle install：

bundler_args: --binstubs

{：data-file =“。travis.yml”}

针对多个版本的依赖项进行测试 #

许多项目需要针对多个版本的Rack，EventMachine进行测试，

HAML，Sinatra，Ruby on Rails等。

要测试多个版本的依赖项：

  1. 在项目的存储库根目录中创建一个目录，您将保留该目录

宝石文件，如./gemfiles。

  1. 添加一个或多个gemfiles。
  2. 设置gemfile关键在你的.travis.yml。

Thoughtbot的Paperclip是[针对多个ActiveRecord进行了测试

版本](https://github.com/thoughtbot/paperclip/blob/master/.travis.yml)：

gemfile:
  - gemfiles/rails2.gemfile
  - gemfiles/rails3.gemfile
  - gemfiles/rails3_1.gemfile

{：data-file =“。travis.yml”}

另一种方法是使用环境变量并制作测试运行器

使用它们。例如，[

](https://github.com/sinatra/sinatra/blob/master/.travis.yml)

env:
  - "rack=1.3.4"
  - "rack=master"
  - "tilt=1.3.3"
  - "tilt=master"

ChefSpec是[

版本](https://github.com/chefspec/chefspec/blob/master/.travis.yml)

env:
  - CHEF_VERSION=14.3.37
  - CHEF_VERSION=13.10.0
  - CHEF_VERSION=12.22.5

引擎，托管服务提供商等。

升级RubyGems #

Travis CI的Ruby环境中安装的RubyGems版本取决于它是什么

由最新的Bundler / RubyGems组合安装，并保持最新

尽可能。

如果您需要最新版本的RubyGems，可以添加以下内容

您的.travis.yml：

before_install:
  - gem update --system
  - gem --version

{：data-file =“。travis.yml”}

要降级到特定版本的RubyGems：

before_install:
  - gem update --system 2.1.11
  - gem --version

{：data-file =“。travis.yml”}

请注意，这将影响您的整体测试时间，作为额外的网络

下载和安装是必需的。
