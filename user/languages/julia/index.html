选择Julia版本进行测试 #

Travis CI的Julia工作人员下载并安装了Julia的二进制文件。你可以指定

朱莉娅版本在测试中julia:关键在你的.travis.yml文件。例如：

language: julia
julia:
  - nightly
  - 1.1
  - 1.0.3

{：data-file =“。travis.yml”}

可接受的格式是：

  • nightly将测试最新的每晚建造

朱莉娅

  • X.Y将针对该次要版本的最新版本进行测试。
  • X.Y.Z将测试该确切的版本。

二进制文件可用的最旧版本是Linux的0.3.1，

或0.2.0表示苹果系统。

覆盖 #

服务如codecov.io和coveralls.io

提供测试套件覆盖范围的摘要和分析。

在为存储库启用相应的服务之后，codecov和coveralls

选项可以如下使用，将它们放在YAML文档的顶层：

codecov: true
coveralls: true

然后，这将在成功完成测试后上传覆盖率统计信息

指定的服务。

默认构建和测试脚本 #

如果您的存储库包含JuliaProject.toml要么Project.toml文件，你是

在Julia v0.7或更高版本上构建，默认的构建脚本将是：

using Pkg
Pkg.build() # Pkg.build(; verbose = true) for Julia 1.1 and up
Pkg.test(coverage=true)

否则它将使用旧表格：

if VERSION >= v"0.7.0-DEV.5183"
    using Pkg
end
Pkg.clone(pwd())
Pkg.build("$pkgname") # Pkg.build("$pkgname"; verbose = true) for Julia 1.1 and up
Pkg.test("$pkgname", coverage=true)

包名称所在的位置$pkgname是存储库名称，包含任何尾随.jl除去。

请注意coverage=true论证只说明Pkg.test发出覆盖信息

关于它运行的测试;它不会将此信息提交给任何服务。

要提交覆盖率信息，请参阅上面的覆盖范围部分。
