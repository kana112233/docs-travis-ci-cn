Objective-C vs Swift #

马上，language: swift只是一个别名language: objective-c。换句话说，我们目前没有对Swift项目的原生支持。 Swift构建只是路由到我们的macOS映像，与Objective-C构建相同。

共享方案 #

为了在Travis CI上运行测试，您还需要创建一个共享

您的应用程序目标的方案，并确保所有依赖项（例如

CocoaPods）明确添加到Scheme中。为此：

  1. 打开管理方案选择表格**产品→方案→

管理方案……**菜单选项。

  1. 在列表中找到要用于测试的目标。确保

共享选中工作表最右侧列中的复选框。

  1. 如果您的目标包含跨项目依赖项，例如CocoaPods，那么

您需要确保将它们配置为显式

依赖。为此：

  1. 突出显示您的应用程序目标并点击编辑…按钮到

打开Scheme编辑表。

  1. 点击建立方案编辑器左侧面板中的选项卡。
  2. 点击+按钮并将每个依赖项添加到项目中。

CocoaPods将显示为名为的静态库**。

  1. 将依赖项拖到测试目标上方，以便首先构建它。

您现在将拥有一个新文件xcshareddata / xcschemes目录

在你的Xcode项目下面。这是你刚才的共享方案

配置。将此文件检入您的存储库，xcodebuild就可以了

找到并执行你的测试。

###

为了能够运行测试，xcodebuild需要知道运行它们的设备，是否

这是一个真正的设备（运行xcodebuild的Mac或连接到Mac的iOS设备）

或模拟器。当您在Travis CI中运行测试时，您需要告诉xcodebuild

您想要通过指定使用的模拟器设备目的地。

设备目标是标识要使用的特定设备的字符串。你可以通过

他们使用的xcodebuild-destination旗。如果您使用的是默认脚本

在您的Travis CI构建中，您可以使用xcode_destination你的.travis.yml中的密钥：

xcode_destination: platform=iOS Simulator,OS=11.3,name=iPhone X

设备目标是以逗号分隔的键值对列表。当你测试

在Travis CI上，您应在设备目的地中包含以下键：

  • platform：其中一个macOS，iOS Simulator，watchOS Simulator，tvOS Simulator。

（“模拟器”部分很重要.Travis CI不支持运行测试

硬件iOS设备）

  • 如果platformmacOS，还包括：
    • OS：模拟设备上OS的版本号。
    • name：模拟设备的名称。例如：“iPhone X”或“Apple TV 1080p”。

有效设备目标的一些示例包括：

  • platform=macOS
  • platform=iOS Simulator,OS=9.3,name=iPhone 5s
  • platform=tvOS Simulator,OS=11.0,name=Apple TV 4K

设备目标在这些设备中唯一标识您的设备非常重要

Xcode知道的。由于Travis CI的构建图像有许多不同的模拟器

安装操作系统版本时，应在设备目标中指定操作系统版本

单独的名称不可能唯一地标识单个模拟器。

您可以在xcodebuild手册页中了解有关设备目标的更多信息。如果你在你的身上

依赖管理 #

Travis CI使用的CocoaPods安装你的项目

依赖。

Travis CI运行的默认命令是：

pod install

请注意，这仅在我们检测项目根目录中的Podfile时运行

目录。如果Podfile位于不同的目录中，则可以使用podfile

设置在.travis.yml：

podfile: path/to/Podfile

{：data-file =“。travis.yml”}

也，pod install如果Pods目录已经销售并且存在，则不会运行

没有更改Podfile.lock文件。

如果有的话Gemfilepod命令是

没有执行，而是使用Bundler作为包装器pod如下：

bundle exec pod install

如果您想使用不同的方法来处理项目的依赖项，

你可以覆盖install命令。

install: make get-deps

{：data-file =“。travis.yml”}
