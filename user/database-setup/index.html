本指南介绍如何在Travis CI环境中设置最常用的数据库和其他服务。

您可以在正在使用的构建环境中检查数据库和服务可用性这里。

除了一些添加的用户和宽松的安全设置外，所有服务都使用默认设置。

启动服务 #

Travis CI环境默认情况下不启动服务，以提供更多RAM

构建脚本。通过将服务添加到服务来启动服务services:你的部分

.travis.yml：

services: mongodb

{：data-file =“。travis.yml”}

如果您在中安装服务addons:部分，例如MariaDB，您不需要将其添加到services:部分也是如此。

要启动多项服务：

services:
  - riak
  - rabbitmq
  - memcached

{：data-file =“。travis.yml”}

如果您手动下载并安装服务，则还必须在a中启动它before_install步。该services密钥仅适用于我们提供的服务。

在你的启动MySQL.travis.yml：

services:

{：data-file =“。travis.yml”}

要求认证。您可以使用用户名进行连接travis要么root

和一个空密码。

MySQL 5.7 #

MySQL 5.7是Xenial映像的默认设置。

在Trusty上，您可以通过添加以下行来安装MySQL 5.7.travis.yml：

addons:
  apt:
    sources:
      - mysql-5.7-trusty
    packages:
      - mysql-server
      - mysql-client

{：data-file =“。travis.yml”}

您还需要将root密码重置为除以外的密码new_password：

before_install:
  - sudo mysql -e "use mysql; update user set authentication_string=PASSWORD('new_password') where User='root'; update user set plugin='mysql_native_password';FLUSH PRIVILEGES;"
  - sudo mysql_upgrade -u root -pnew_password
  - sudo service mysql restart

{：data-file =“。travis.yml”}

PostgreSQL的 #

在你的启动PostgreSQL.travis.yml：

services:
  - postgresql

{：data-file =“。travis.yml”}

在Builds中使用PostgreSQL #

访问本地PostgreSQL服务器的默认用户是postgres密码为空。

通过在.travis.yml中添加一行来为您的应用程序创建数据库：

before_script:
  - psql -c 'create database travis_ci_test;' -U postgres

{：data-file =“。travis.yml”}

对于Rails应用程序，您现在可以使用以下内容database.yml在本地访问数据库的配置：

test:
  adapter: postgresql
  database: travis_ci_test

{：data-file =“database.yml”}

如果您的本地测试设置使用不同的凭据或设置来访问本地测试数据库，我们建议将这些设置放在一个database.yml.travis

before_script:
  - cp config/database.yml.travis config/database.yml

{：data-file =“。travis.yml”}

使用PostGIS #

安装与PostgreSQL版本匹配的PostGIS版本，并使用以下命令激活PostGIS扩展：

addons:

    - postgresql-9.6-postgis-2.3
before_script:
  - psql -U postgres -c "create extension postgis"

{：data-file =“。travis.yml”}

Travis CI构建环境附带了许多预安装的语言环境，但如果需要，您还可以安装其他语言环境。

安装区域设置 #

以下示例显示了您需要添加到您的行.travis.yml安装西班牙语语言包。

sqlite3的 #

最简单，最简单的关系数据库。

Ruby项目中的SQLite3 #

将sqlite3 ruby​​绑定添加到您的包中：

# Gemfile
# for CRuby, Rubinius, including Windows and RubyInstaller
gem "sqlite3", :platform => [:ruby, :mswin, :mingw]

# for JRuby
gem "jdbc-sqlite3", :platform => :jruby

如果您使用ActiveRecord，请将以下内容添加到您的config/database.yml：

test:
  adapter: sqlite3
  database: ":memory:"
  timeout: 500

{：data-file =“config / database.yml”}

或者，如果你没有使用config/database.yml，手动连接到数据库：

ActiveRecord::Base.establish_connection :adapter => 'sqlite3',
                                        :database => ':memory:'

MongoDB的 #

在你的启动中启动MongoDB.travis.yml：

services:
  - mongodb

{：data-file =“。travis.yml”}

MongoDB绑定到127.0.0.1并且不需要预先进行身份验证或数据库创建。如果你添加一个admin自启用用户身份验证mongod是从…开始的--auth论点。

注意：管理员用户是在管理数据库中创建的用户。

要为数据库创建用户，请添加before_script你的部分.travis.yml：

before_script:
  - mongo mydb_test --eval 'db.createUser({user:"travis",pwd:"test",roles:["readWrite"]});'

{：data-file =“。travis.yml”}

MongoDB不会立即接受连接 #

一些用户报告说，MongoDB在构建脚本时不接受连接。

问题是间歇性的，唯一可靠的方法是避免它

在进行第一次连接之前注入人工等待：

添加以下内容before_script到你的.travis.yml在连接到MongoDB之前等待：

before_script:
  - sleep 15
  - mongo mydb_test --eval 'db.createUser({user:"travis",pwd:"test",roles:["readWrite"]});'

CouchDB的 #

.travis.yml：

services:
  - couchdb

CouchDB绑定到127.0.0.1，使用默认配置，不需要身份验证（在CouchDB中，它在管理员方面运行）。

在使用CouchDB之前，您需要在构建过程中创建数据库：

before_script:
  - curl -X PUT localhost:5984/myapp_test

{：data-file =“。travis.yml”}

ElasticSearch #

在你的启动中启动ElasticSearch.travis.yml：

services:
  - elasticsearch

{：data-file =“。travis.yml”}

ElasticSearch需要几秒钟才能启动，以确保在构建脚本运行时它可用于为构建脚本添加一个小延迟：

before_script:
  - sleep 10

ElasticSearch使用默认配置，可在127.0.0.1上使用。

安装特定版本的ElasticSearch #

您可以使用以下所需的版本覆盖已安装的ElasticSearch（例如，2.3.0）：

before_install:
  - curl -O https://download.elastic.co/elasticsearch/release/org/elasticsearch/distribution/deb/elasticsearch/2.3.0/elasticsearch-2.3.0.deb && sudo dpkg -i --force-confnew elasticsearch-2.3.0.deb && sudo service elasticsearch restart

{：data-file =“。travis.yml”}

我们建议验证下载URL的有效性在ElasticSearch的网站上。

sudo不可用基于容器的基础设施。

###

默认情况下，未安装ElasticSearch可靠的基于容器的基础设施

但你可以通过添加以下步骤来安装它.travis.yml。

env:
  - ES_VERSION=5.1.1 ES_DOWNLOAD_URL=https://artifacts.elastic.co/downloads/elasticsearch/elasticsearch-${ES_VERSION}.tar.gz
install:
  - wget ${ES_DOWNLOAD_URL}
  - tar -xzf elasticsearch-${ES_VERSION}.tar.gz
  - ./elasticsearch-${ES_VERSION}/bin/elasticsearch &
script:
  - wget -q --waitretry=1 --retry-connrefused -T 10 -O - http://127.0.0.1:9200

{：data-file =“。travis.yml”}

RethinkDB #

要将RethinkDB与Travis CI一起使用，请将其列为插件中的插件.travis.yml配置文件，将版本号指定为字符串。

addons:
  rethinkdb: '2.3.4'

{：data-file =“。travis.yml”}

如果指定部分版本号，则插件将安装并运行匹配的最新版本。例如，'2.3'将匹配最新的RethinkDB版本2.3.x线。

导出两个环境变量：

  • TRAVIS_RETHINKDB_VERSION是配置中指定的版本（例如，'2.3.4'， 要么'2.3'）。
  • TRAVIS_RETHINKDB_PACKAGE_VERSION是已安装的软件包的完整版本（例如，'2.3.4+1~0precise'）。

启用后，RethinkDB将启动localhost在默认端口（28015）。

多个数据库构建 #

如果需要使用不同的数据库运行多个构建，则可以配置环境变量

和abefore_script要么before_install用于创建构建矩阵的行。

使用环境变量和before_script步骤 #

使用DB环境变量，用于指定数据库配置的名称。在本地你会运行：

DB=postgres [commands to run your tests]

在Travis CI上你想要创建一个构建矩阵每个都有三个版本DB变量导出具有不同的值，并为此可以使用env选项.travis.yml：

env:
  - DB=sqlite
  - DB=mysql
  - DB=postgres

{：data-file =“。travis.yml”}

然后你可以在一个中使用这些值before_install（要么before_script）步骤

设置每个数据库。例如：

before_script:
  - sh -c "if [ '$DB' = 'postgres' ]; then psql -c 'DROP DATABASE IF EXISTS tests;' -U postgres; fi"
  - sh -c "if [ '$DB' = 'postgres' ]; then psql -c 'DROP DATABASE IF EXISTS tests_tmp;' -U postgres; fi"
  - sh -c "if [ '$DB' = 'postgres' ]; then psql -c 'CREATE DATABASE tests;' -U postgres; fi"
  - sh -c "if [ '$DB' = 'postgres' ]; then psql -c 'CREATE DATABASE tests_tmp;' -U postgres; fi"
  - sh -c "if [ '$DB' = 'mysql' ]; then mysql -e 'CREATE DATABASE IF NOT EXISTS tests_tmp; CREATE DATABASE IF NOT EXISTS tests;'; fi"

{：data-file =“。travis.yml”}

Travis CI对这些变量没有任何特殊支持

创建具有不同导出值的三个构建。这取决于你的构建

脚本和before_install要么before_script使用它们的步骤。

有关一个真实的例子，请参阅[学说/ doctrine2

.travis.yml](https://github.com/doctrine/doctrine2/blob/master/.travis.yml)。

使用Ruby #

另一种方法是将所有数据库配置放在一个YAML文件中（test/database.yml

sqlite:
