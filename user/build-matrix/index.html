工作。

## 不包括工作

您还可以定义构建矩阵的排除项：

matrix: exclude:

  • rvm: 1.9.3 gemfile: gemfiles/Gemfile.rails-2.3.x env: ISOLATED=true
  • rvm: jruby gemfile: gemfiles/Gemfile.rails-2.3.x env: ISOLATED=true

{：data-file =“。travis.yml”}

如果要从构建矩阵中排除的作业共享相同的矩阵

参数，您只能指定那些并省略不同的部分。

假设你有：

language: ruby rvm:

  • 1.9.3
  • 2.0.0
  • 2.1.0 env:
  • DB=mongodb
  • DB=redis
  • DB=mysql gemfile:
  • Gemfile
  • gemfiles/rails4.gemfile
  • gemfiles/rails31.gemfile
  • gemfiles/rails32.gemfile

{：data-file =“。travis.yml”}

这导致3×3×4构建矩阵。排除所有有的工作`rvm`值`2.0.0`和

`gemfile`值`Gemfile`， 你可以写：

matrix: exclude:

  • rvm: 2.0.0 gemfile: Gemfile

{：data-file =“。travis.yml”}

这相当于：

matrix: exclude:

  • rvm: 2.0.0 gemfile: Gemfile env: DB=mongodb
  • rvm: 2.0.0 gemfile: Gemfile env: DB=redis
  • rvm: 2.0.0 gemfile: Gemfile env: DB=mysql

{：data-file =“。travis.yml”}

### 不包括工作`env`### 值

在排除职位时`env`值，值必须匹配

究竟。

例如，

language: ruby rvm:

  • 1.9.3
  • 2.0.0
  • 2.1.0 env:
  • DB=mongodb SUITE=all
  • DB=mongodb SUITE=compact
  • DB=redis
  • DB=mysql matrix: exclude:
    • rvm: 1.9.3 env: DB=mongodb

{：data-file =“。travis.yml”}

定义了一个3×4矩阵，因为`env`值与之不匹配

矩阵中定义的任何工作。

使用排除所有Ruby 1.9.3作业`DB=mongodb`设置，写：

language: ruby rvm:

  • 1.9.3
  • 2.0.0
  • 2.1.0 env:
  • DB=mongodb SUITE=all
  • DB=mongodb SUITE=compact
  • DB=redis
  • DB=mysql matrix: exclude:
    • rvm: 1.9.3 env: DB=mongodb SUITE=all # not ‘env: DB=mongodb SUITE=all’ or ‘env: SUITE=all DB=mongodb’
    • rvm: 1.9.3 env: DB=mongodb SUITE=compact # not ‘env: SUITE=compact DB=mongodb’

{：data-file =“。travis.yml”}

## 明确包括工作

也可以在矩阵中包含条目`matrix.include`：

matrix: include:

  • rvm: ruby-head gemfile: gemfiles/Gemfile.rails-3.2.x env: ISOLATED=false

{：data-file =“。travis.yml”}

这会将特定作业添加到已填充的构建矩阵中。

如果您只想测试最新版本的依赖项以及最新版本的运行时，这非常有用。

您可以使用此方法创建仅包含特定组合的构建矩阵。

例如，

language: python matrix: include:

  • python: “2.7” env: TEST_SUITE=suite_2_7
  • python: “3.3” env: TEST_SUITE=suite_3_3
  • python: “pypy” env: TEST_SUITE=suite_pypy script: ./test.py $TEST_SUITE

{：data-file =“。travis.yml”}

创建一个包含3个作业的构建矩阵，为每个版本运行测试套件

Python

### 显式包含的作业继承数组中的第一个值

显式包含的作业将继承扩展的第一个值

键定义。

在这个带有3个作业的Python构建矩阵的示例中，每个作业都在`matrix.include`

有`python`值设置为`'3.5'`。

您可以为特定条目显式设置python版本：

language: python python:

  • ‘3.5’
  • ‘3.4’
  • ‘2.7’ matrix: include:
    • python: ‘3.5’ # this is not strictly necessary env: EXTRA_TESTS=true
    • python: ‘3.4’ env: EXTRA_TESTS=true script: env $EXTRA_TESTS ./test.py $TEST_SUITE

{：data-file =“。travis.yml”}

## 允许失败的行

您可以在构建矩阵中定义允许失败的行。允许

失败是构建矩阵中允许失败但不会导致的项目

整个构建失败。这可以让你添加实验和

准备版本以测试您不是的版本或配置

准备正式支持。

将构建矩阵中允许的失败定义为键/值对：

matrix: allow_failures:

  • rvm: 1.9.3

{：data-file =“。travis.yml”}

### 匹配工作与`allow_failures`

将作业与给定的定义匹配时`allow_failures`，所有

条件`allow_failures`必须完全满足，并且

所有的钥匙`allow_failures`元素必须存在于

构建矩阵的顶层（即不在`matrix.include`）。

`allow_failures`#### 例子

考虑

language: ruby

rvm:

  • 2.0.0
  • 2.1.6

env: global:

  • SECRET_VAR1=SECRET1 matrix:
  • SECRET_VAR2=SECRET2

matrix: allow_failures: - env: SECRET_VAR1=SECRET1 SECRET_VAR2=SECRET2


{：data-file =“。travis.yml”}

在这里，没有工作可以失败，因为没有工作`env`值

`SECRET_VAR1=SECRET1 SECRET_VAR2=SECRET2`。

下一个，

language: php php:

  • 5.6
  • 7.0 matrix: include:
    • php: 7.0 env: KEY=VALUE allow_failures:
    • php: 7.0 env: KEY=VALUE

{：data-file =“。travis.yml”}

没有顶级`env`，任何工作都不会失败。

## 快速整理

如果允许构建矩阵中的某些行失败，则构建在完成之前不会标记为已完成。

`fast_finish: true``matrix`你的部分`.travis.yml`

matrix: fast_finish: true

```

现在，一旦所有必需的工作完成，将根据这些结果确定构建结果，而其余部分allow_failures工作继续运行。
